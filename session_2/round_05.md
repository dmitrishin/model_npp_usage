## Раунд 5 — Предварительное голосование (ТОП-20)

---

Место: 1
ID идеи: R1-1-02
Идея: Управление критическим путём тепломонтажных работ (ТМР) и сварки трубопроводов в основном периоде — DAG с вероятностными длительностями, GERT для альтернативных последовательностей монтажа, ML для прогнозирования длительности сварки, многомерная оптимизация (срок, стоимость, риск брака, ресурсы сварщиков).
Почему: ТМР и сварка трубопроводов — единственный общепризнанный (всеми участниками и материалами) определяющий критический путь строительства АЭС. Отставание ТМР — главный фактор задержек на всех проектах. Модель здесь работает на самой болезненной производственной проблеме с наибольшей потенциальной отдачей. Идея не получила критических замечаний, опровергающих её реализуемость.

Место: 2
ID идеи: R1-1-05
Идея: Количественный анализ рисков графика и стоимости (QRA) в соответствии с требованиями ТСМ NC — PERT + Монте-Карло для распределения сроков и стоимости, критерии принятия решений для утверждения базовых линий, ML для прогнозирования рисков, анализ чувствительности.
Почему: QRA — прямое требование действующего отраслевого стандарта ТСМ NC (5 этапных оценок до первого бетона). Модель Дмитришина предоставляет полный математический аппарат для этой задачи. Бизнес-ценность немедленна: обоснование Целевой и Предельной стоимости, определение резервов. Внедрение не требует революционных изменений — встраивается в существующий регламент.

Место: 3
ID идеи: R1-1-01
Идея: Оптимизация и прогнозирование интегрированного комплексного графика (ИКГ) — графика 2-го уровня. После доработки по итогам критики: модель как инструмент подготовки обоснования для CCB, не автономная система; обязательный этап очистки данных; метрика уверенности.
Почему: ИКГ — центральный документ управления проектом. Необоснованные графики с задержками 100–200% — системная проблема отрасли. Доработка по итогам критики (C-3-01, C-4-01) существенно усилила идею, позиционировав модель как DSS для процедуры утверждения, а не как замену процедуры. Высочайшая производственная ценность.

Место: 4
ID идеи: R1-1-03
Идея: Оптимизация графика подготовительного периода с прогнозированием влияния на стартовые работы основного периода — единый граф ПП с GERT-развилками, Монте-Карло для ключевых событий, многомерная оптимизация баланса вложений.
Почему: Качество подготовительного периода определяет успех всего проекта. Длительность ПП (2–8 лет) и множество параллельных потоков (лицензирование, проектирование, контрактация, площадка) создают сложнейшую задачу координации. Модель позволяет прогнозировать влияние задержек ПП на основной период ещё до утверждения графиков 2–3 уровней.

Место: 5
ID идеи: R1-1-06
Идея: Оптимизация последовательности и сроков ПНР с обратным планированием от даты ввода — граф ПНР с обратным ходом, GERT для условных переходов (решения ГРП), Монте-Карло для определения требуемых дат готовности систем, многомерная оптимизация (временные vs постоянные схемы).
Почему: ПНР — финальный и наиболее рискованный этап, напрямую определяющий дату ввода. Обратное планирование от даты ввода через этапы ПНР задаёт жёсткие требования к срокам завершения монтажа и поставок. Идея усилена аналогичными предложениями от участников 2, 3, 4, 5, 6, 7 — универсальное признание важности.

Место: 6
ID идеи: R1-6-01
Идея: Многокритериальная оптимизация стратегии контрактации ОДЦИ на базе GERT-сетей и ML — GERT с циклами возврата документации и срывами тендеров, ML для прогнозирования реальных сроков по истории поставщиков, критерий Сэвиджа для выбора поставщика в пространстве (Время, Стоимость, Риск).
Почему: Отлично проработанный механизм с конкретной привязкой к бизнес-процессу Росатома. Доработка по критике (C-4-06): при малой статистике — переключение на Вальд. Обеспечивает поставку по принципу just-in-time, защищая критический путь и оптимизируя складские площади СМБ.

Место: 7
ID идеи: R1-1-07
Идея: Оптимизация стратегии инвестирования в подготовительный период — формализация сценариев, критерии Вальда/Сэвиджа/Гурвица, Парето-фронт «стоимость ПП — потери в ОП — риск потерь при отмене». По доработке: модель структурирует выбор ЛПР, а не претендует на объективный ответ; критерии Вальд/Сэвидж работают без вероятностей.
Почему: Фундаментальное стратегическое решение о структуре финансирования ПП. Доработка после собственной критики (C-1-11) честно признала ограничения и переформулировала ценность: прозрачность структуры выбора, а не иллюзия объективности. Это усилило идею.

Место: 8
ID идеи: R1-1-14
Идея: Construction Readiness Review — формализация критериев перехода к основному периоду. По доработке: CRR с независимыми фактическими индикаторами + вероятностный прогноз; расхождение между прогнозом и фактом — сигнал о системном оптимизме.
Почему: Формальный «шлюз» перед первым бетоном — международная лучшая практика (SRS-119, NEA 7530, IAEA NG-T-1.6). Доработка по критике C-2-01 (самоподтверждающийся контур) существенно усилила идею. Обоснованное решение go/no-go предотвращает старт с незрелым проектом.

Место: 9
ID идеи: R1-1-04
Идея: Оптимизация опережающей контрактации оборудования — формализация цепочки «ИТТ → конкурс → контракт → изготовление → поставка → монтаж», Монте-Карло для вероятности поставки к дате, GERT для альтернативных стратегий контрактации, итеративная оптимизация перечня.
Почему: Задержки поставок — один из главных факторов отставания ТМР и электромонтажных работ на всех проектах. Три волны контрактации в ПП по стандарту — прямая область применения модели. Идея охватывает оба периода.

Место: 10
ID идеи: R1-1-08
Идея: Управление графиком выпуска РД с синхронизацией с графиками СМР и поставок — граф выпуска РД с зависимостями между комплектами и циклами получения ИДП, GERT для ситуаций многократных ревизий, ML для прогнозирования длительности выпуска, итеративная оптимизация приоритетов.
Почему: Задел 40–50% РД к первому бетону — обязательное условие. Темп проектирования зависит от сложных взаимозависимостей между дисциплинами и поставщиками. Модель позволяет предотвратить ситуацию, когда СМР не обеспечены документацией — критическую и повсеместную проблему.

Место: 11
ID идеи: R1-1-20
Идея: Адаптивное управление проектом — обновление модели по мере поступления фактических данных. По доработке: переход Beta→эмпирика с обязательной стратификацией; уникальные операции остаются на PERT; ML переобучается на фактических данных с метрикой уверенности.
Почему: Обеспечивает жизнеспособность модели на всём протяжении проекта (7–12+ лет). Доработка по критике (C-4-09, C-7-01, C-6-09) — стратификация и дифференцированный подход — существенно усилила идею, устранив методологическую слабость.

Место: 12
ID идеи: R1-2-16
Идея: Расчёт адресных «пакетов резервов» по сроку, стоимости и ресурсам — не один общий буфер, а набор адресных резервов по узлам с наибольшей чувствительностью.
Почему: Решает конкретную управленческую проблему: общий буфер неэффективен — либо избыточен, либо недостаточен на конкретных участках. Адресные резервы, распределённые по результатам анализа чувствительности, — практичный и реализуемый механизм. Прямая связь с QRA (место 2).

Место: 13
ID идеи: R1-1-10
Идея: Оптимизация поточного строительства многоблочной АЭС — определение оптимального шага ввода с учётом общих ресурсов, эффекта обучения, финансовой модели. По доработке: внешние ограничения (контрактные обязательства системного оператора, макроэкономика) — жёсткие граничные условия.
Почему: Серийное строительство — стратегический приоритет Росатома. Обоснование шага ввода (12/18/24 месяца) определяет экономику всей программы. Доработка по критике (C-8-01) корректно позиционирует модель внутри внешних ограничений.

Место: 14
ID идеи: R1-7-07
Идея: Системная оценка проектных изменений (Change Control) — многомерная оценка влияния изменения на стоимость, сроки, трудозатраты; ML-классификация риска по базе Lessons Learned. По доработке: ML-фильтр для мелких изменений (fast-track), полная симуляция только для каскадно значимых.
Почему: Неконтролируемые проектные изменения — главный источник удорожания по международному опыту (NEA 7530). Доработка по критике (C-8-05) решила проблему перегрузки процедуры — дифференциация по значимости делает идею практически применимой.

Место: 15
ID идеи: R2-1-03
Идея: Определение оптимального момента фиксации внутренней проектной базы с оценкой остаточного риска внешних регуляторных изменений. По доработке: не «заморозка требований регулятора», а внутренняя фиксация командой проекта; внешние изменения — через формальное управление изменениями.
Почему: Принцип заморозки проекта — проверенная практика (СССР, конференция 1980 г., NEA 7530). Доработка по критике (C-2-08, C-7-05) корректно разграничила внутренний и внешний контуры. Ключевой фактор предотвращения «ползучих изменений».

Место: 16
ID идеи: R1-1-11
Идея: Обнаружение аномалий и раннее предупреждение о рисках отклонений от графика — обновление распределений по факту, ML-детекция аномалий (Autoencoder, Isolation Forest), кластеризация для выявления системных проблем. По доработке (аналог R1-8-05): недельный такт, привязка к существующим источникам (P6, 1С-КСУ).
Почему: Раннее выявление отклонений предотвращает каскадные задержки. Доработка от участника 8 (недельный такт, существующие источники) делает идею реализуемой без новой инфраструктуры. Прямо дополняет идеи по управлению критическим путём (место 1).

Место: 17
ID идеи: R1-6-05
Идея: Моделирование возвратов при передаче систем в ПНР с помощью GERT-циклов и ML-кластеризации — стохастическая GERT-сеть с вероятностями возврата системы на доработку, DBSCAN для кластеризации подрядчиков по частоте брака, Байесовский критерий для буферов времени.
Почему: Переход от СМР к ПНР — один из самых проблемных стыков в строительстве АЭС. Возвраты систем строителям на устранение дефектов — типичная и плохо учитываемая в графиках реальность. GERT-циклы — идеальный инструмент для моделирования этого нелинейного процесса.

Место: 18
ID идеи: R1-1-12
Идея: Оптимизация графика передачи помещений и координации боксового монтажа — локальные подграфы для каждого помещения с переходами между специальностями, GERT для условий передачи, многомерная оптимизация на уровне пускового комплекса с координацией поставок.
Почему: Организация работ в помещениях по потокам (отделка → тепломонтаж → электромонтаж → вентиляция) — основа поточного метода строительства. Несвоевременная передача помещений — частая причина конфликтов и простоев. Практическая, реализуемая идея с непосредственной производственной ценностью.

Место: 19
ID идеи: R2-1-02
Идея: Прогнозирование и минимизация объёма бросовых и двойных работ при изменении решений. По доработке: добавлен атрибут «регуляторные последствия» (повторная экспертиза, лицензия); решения, затрагивающие ПОС и ООБ, получают флаг повышенного регуляторного риска; критерий Вальда для таких решений.
Почему: Бросовые работы — прямые убытки и бесполезная нагрузка. Доработка по критике (C-3-05) существенно усилила идею, включив юридические последствия в целевую функцию. Применимо в подготовительном периоде при выборе стратегии опережающего проектирования.

Место: 20
ID идеи: R1-1-09
Идея: Прогнозирование и оптимизация процесса лицензирования АЭС — DAG лицензионного процесса, GERT для циклов доработки по замечаниям регулятора, Монте-Карло для оценки вероятности получения лицензии к целевой дате, критерии решений для стратегии параллельного ведения работ.
Почему: Лицензирование — на критическом пути любого проекта (7–9 лет в РФ, 3–9 лет международно). Модель позволяет количественно оценить неопределённость и выбрать стратегию параллельного ведения работ. Применимо как к российским, так и к экспортным проектам.
