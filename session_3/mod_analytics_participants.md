## 12. Ключевые выводы

**1. Глубина проработки > количество.** У6 с 10 идеями дал 3 финалиста (30%), У8 с 92 идеями — 0. Развёрнутое обоснование и описание механизма критичны для привлечения голосов.

**2. Критика — самостоятельная форма вклада.** У3 — 0 идей в финале, но 3 критики привели к доработкам финалистов и 3 доработки чужих идей в R4. Без C-3-03 идея R1-1-03 не получила бы двухрежимную модель и, вероятно, не заняла бы 1-е место.

**3. Объективность голосования высока.** У4, У5, У8 голосовали на 100% за чужие идеи. Среднее по всем участникам — 86% чужих голосов в R6. Авторская предвзятость минимальна.

**4. Точность считывания приоритетов различается.** У5 — 9/10 совпадений с ТОП-10 (лучший результат). У2, У4, У6, У7 — 7/10 (наибольшее расхождение с консенсусом).

**5. Парные модели одного разработчика показали разные профили.** Claude Opus 4.6 (У1) и Claude Opus 4.5 (У8): один — лидер по авторству и доработке, другой — по объёму с нулевой конверсией. Gemini Deep Think (У6) и Gemini Studio (У7): один — «снайпер» с 30% конверсией, другой — «резонатор» с фокусом на поддержке сильных чужих идей.

**6. Наиболее полный вклад — у моделей, активных на всех этапах.** У1 (генерация + критика + доработка + голосование) и У6 (точная генерация + качественная критика + доработка) — два самых результативных участника с разными стратегиями.
