## Раунд 2 — Новые идеи

**ID: R2-1-01**
**Идея:** Применение модели для прогнозирования процента брака сварных соединений и его каскадного влияния на график ТМР и подготовку к ХГО.
**Обоснование:** Сварка трубопроводов — критический путь основного периода. Определённый процент сварных соединений не проходит неразрушающий контроль (НК) и подлежит ремонту или вырезке. Этот процент варьируется по бригадам, типам соединений, условиям. Механизм: ML-модели (XGBoost, CatBoost) обучаются на накопленных данных НК предсказывать вероятность отбраковки конкретного стыка по признакам (тип шва, марка стали, квалификация сварщика, температура, предшествующие результаты); GERT-ветвления моделируют исходы контроля (годен / ремонт / вырезка) с разными длительностями последствий; Монте-Карло пересчитывает влияние на график ТМР и дату готовности к ХГО; локальная оптимизация перераспределяет бригады и назначения так, чтобы стыки с высокой прогнозной вероятностью брака выполнялись лучшими бригадами. Отличается от идей управления критическим путём ТМР тем, что механизм решения работает на уровне предиктивного управления качеством конкретных технологических операций, а не на уровне перепланирования последовательности работ.

**ID: R2-1-02**
**Идея:** Применение модели для оптимизации стратегии взаимодействия с регулятором по форме и последовательности подачи лицензионных документов (комплексная подача vs. поэтапная, опережающий диалог vs. формальная процедура).
**Обоснование:** Идеи Раунда 1 моделируют процесс лицензирования как данность (GERT-сеть с вероятностями). Здесь механизм другой: модель оптимизирует саму стратегию подачи документов. Альтернативные стратегии (подать весь пакет сразу vs. подавать поэтапно по разделам; провести предварительные консультации vs. формальная подача; задействовать институт научно-технической поддержки vs. прямое взаимодействие) представляются как разные траектории в GERT. Для каждой стратегии ML на данных прошлых лицензионных процессов оценивает вероятности количества итераций и их длительность. Многокритериальная оптимизация (T, C, R) выбирает стратегию, минимизирующую общую длительность лицензирования при заданном бюджете на подготовку документов. Критерий Сэвиджа применяется для минимизации сожаления о выбранной стратегии. Даёт лицензионному блоку количественное обоснование подхода к взаимодействию с регулятором.

**ID: R2-1-03**
**Идея:** Применение модели для оптимизации графика проведения инспекций строительного контроля и технического надзора, синхронизированного с ходом СМР.
**Обоснование:** Инспекции Ростехнадзора, строительного контроля заказчика и авторского надзора генпроектировщика требуют готовности объекта и документации, а их результаты (замечания, предписания, остановки работ) создают непредсказуемые задержки. В Раунде 1 инспекции упомянуты только в 8-65 как отдельная узкая тема. Предлагаемый механизм системнее: инспекции встраиваются как обязательные GERT-узлы на критическом пути; вероятности переходов (пройдено без замечаний / с замечаниями / с остановкой работ) калибруются по данным предыдущих проектов; ML классифицирует типы работ по вероятности получения замечаний; Монте-Карло оценивает влияние инспекций на общий срок; локальная оптимизация определяет оптимальные моменты вызова инспекторов — достаточно рано для обнаружения проблем, но не настолько рано, чтобы создать «бутылочное горлышко». Снижает задержки от инспекций, обеспечивая их проведение в оптимальные моменты.

**ID: R2-1-04**
**Идея:** Применение модели как инструмента обучения и тренировки руководителей проектов (project managers) через симуляцию сценариев строительства АЭС.
**Обоснование:** Ни одна идея Раунда 1 не предлагает использовать модель для обучения людей. Механизм: модель работает в режиме симулятора, где обучаемый принимает решения (о распределении ресурсов, реагировании на отклонения, выборе подрядчиков, изменении последовательности работ), а модель генерирует вероятностные последствия через Монте-Карло и GERT-ветвления. ML-компонент настраивает уровень сложности сценариев на основе типичных и нетипичных ситуаций из данных реальных проектов. Деревья решений визуализируют выбор с оценкой по критериям (Вальд для безопасности, Байес для стоимости). Обучаемый видит распределения последствий своих решений. Решает проблему дефицита квалифицированных руководителей проектов АЭС — каждый новый РП может «пройти» несколько виртуальных строек до выхода на реальный объект, накапливая интуицию принятия решений в условиях неопределённости.

**ID: R2-1-05**
**Идея:** Применение модели для оптимизации последовательности испытаний и сдачи технологических систем на этапах А-0/А-1 ПНР с учётом межсистемных зависимостей.
**Обоснование:** На этапах А-0 и А-1 десятки технологических систем проходят индивидуальные испытания и ПНР. Системы взаимозависимы: испытание одной требует работоспособности других (электроснабжение → вентиляция → контрольные системы → технологическая система). Ошибочная последовательность приводит к ожиданиям и простоям. В Раунде 1 ПНР рассматриваются как укрупнённые этапы (А-0 → Г). Механизм здесь — модель работает на уровне отдельных систем внутри этапа А-0/А-1: PERT/GERT-сеть описывает граф межсистемных зависимостей (>200 систем с условиями приёмки); GERT-ветвления моделируют результат испытания (годен / замечания / повторные испытания); Монте-Карло оценивает распределение длительности всей программы испытаний; итеративная оптимизация ищет последовательность, минимизирующую суммарное время простоев пусконаладочного персонала; ML предсказывает вероятность успеха испытания системы по характеристикам предшествующего монтажа. Даёт ГРП (Группе руководства пуском) обоснованный план приоритетов испытаний.

**ID: R2-1-06**
**Идея:** Применение модели для оптимизации управления запасами строительных материалов на площадке (бетон, арматура, металлоконструкции) с учётом стохастического спроса.
**Обоснование:** Ни одна идея Раунда 1 не адресует управление запасами материалов как отдельную задачу. Оборудование (ОДЦИ) покрыто, но расходные строительные материалы — нет. Механизм: потребность в материалах привязана к работам PERT-сети и наследует их вероятностный характер; Монте-Карло генерирует распределение потребности по времени; GERT-ветвления моделируют сценарии поставки (задержка поставщика, сезонные ограничения на перевозки, изменение объёмов работ); многокритериальная оптимизация ищет баланс между размером запаса на площадке (затраты на хранение, площади) и риском остановки работ из-за дефицита; ML прогнозирует фактическое потребление на основе темпов строительства; критерий Вальда применяется для материалов, дефицит которых остановит критический путь (бетон в период массового бетонирования). Решает проблему перебоев с материалами, которая на практике часто является скрытой причиной снижения производительности.

**ID: R2-1-07**
**Идея:** Применение модели для оценки влияния климатических и сезонных факторов на график строительства конкретной площадки и оптимизации сезонного распределения работ.
**Обоснование:** В Раунде 1 климат упоминается лишь фрагментарно (погода при монтаже кранов, подготовка котлована). Предлагаемый механизм — системный: для конкретной площадки (тропический, арктический, умеренный климат, сезон дождей, мерзлота) строится слой климатических ограничений поверх PERT/GERT-сети: бета-распределения длительностей работ модулируются сезонными факторами (зимнее бетонирование, ветровые ограничения для крановых операций, дождливый сезон для земляных работ); ML обучается на метеоданных площадки и фактической производительности прошлых проектов в аналогичных условиях; Монте-Карло с климатическим наложением генерирует сезонно-зависимые прогнозы; многокритериальная оптимизация определяет оптимальное распределение типов работ по сезонам (земляные — в сухой сезон, внутренние монтажные — в дождливый). Особенно ценно для зарубежных площадок с экстремальным климатом (Египет — жара, Бангладеш — муссоны, Турция — сейсмика и зимы).

**ID: R2-1-08**
**Идея:** Применение модели для оптимизации процесса передачи систем и объектов от строительного подрядчика к эксплуатирующей организации в ходе подэтапов ПНР.
**Обоснование:** Передача систем — двусторонний процесс, где строительная сторона должна завершить работы и оформить документацию, а эксплуатирующая организация — принять и начать ПНР. Рассогласование порождает формальные задержки (неготовность актов, недокомплект документации, отсутствие приёмочных комиссий). В Раунде 1 этот организационный переход не адресован как отдельная задача. Механизм: GERT-сеть описывает процесс передачи с условными переходами (замечания приёмочной комиссии → доработка → повторная приёмка); каждой системе приписывается вероятность приёмки с первого раза; ML обучается на данных о типичных замечаниях по типам систем; Монте-Карло оценивает совокупную задержку от процессов передачи; итеративная оптимизация определяет приоритеты подготовки документации и устранения замечаний; деревья решений формализуют выбор — принять с замечаниями и устранить в ходе ПНР или дождаться полного устранения. Снижает потери времени на организационно-документальных интерфейсах между строительной и пусковой командами.

**ID: R2-1-09**
**Идея:** Применение модели для прогнозирования и оптимизации использования тяжёлой строительной техники (башенные краны, бетононасосы, экскаваторы) с учётом стохастических простоев и конфликтов доступа.
**Обоснование:** Тяжёлая техника — дефицитный ресурс, конкурирующий между объектами. Простои из-за поломок, погоды, конфликтов доступа не прогнозируются детерминированными графиками. В Раунде 1 краны упомянуты (8-20, R1-7-03), но как частный случай конкретной монтажной операции, а не как системный ресурс всей стройки. Механизм: техника вводится в модель как ресурсное ограничение на дугах PERT/GERT-сети; ML предсказывает вероятность простоя техники (по данным телеметрии, истории обслуживания, прогнозу погоды); GERT моделирует альтернативы при недоступности техники (ожидание, замена, перенос монтажа); Монте-Карло оценивает влияние на график; многокритериальная оптимизация строит расписание использования техники, минимизирующее суммарные простои при заданном парке. Решает проблему скрытых потерь производительности из-за конкуренции за технику и её незапланированных простоев.

**ID: R2-1-10**
**Идея:** Применение модели для количественного обоснования решений о досрочном завершении подготовительного периода — переход к основному периоду с осознанным принятием определённого объёма незавершённости.
**Обоснование:** На практике ни один подготовительный период не завершается со 100% готовностью. Руководство принимает решение начать основной период с некоторой долей незавершённых задач (не вся РД, не полная контрактация, частичная готовность СМБ). Идеи Раунда 1 о «шлюзах готовности» (R1-1-22, R1-2-01, R1-2-07, R1-5-14) фокусируются на оценке готовности как бинарном Go/No-Go. Механизм здесь другой — модель количественно оценивает последствия перехода при разных уровнях незавершённости: каждый незавершённый элемент (пакет РД, контракт, объект СМБ) вносит свой вклад в удорожание и удлинение основного периода; Монте-Карло моделирует распределение дополнительных затрат и задержек для каждого уровня незавершённости; многокритериальная оптимизация строит фронт Парето: «потери от переноса первого бетона» vs. «потери от начала с незрелым проектом»; критерий Гурвица с настраиваемым α позволяет выбрать компромисс в зависимости от аппетита к риску. Даёт руководству не «готов / не готов», а карту последствий: «при переходе сейчас — ожидаемый рост стоимости ΔC и сроков ΔT с заданной вероятностью».

Комментарий участника: Без комментария.

## Раунд 2 — Новые идеи

ID: R2-2-01
Идея: Применить модель для управления внешним сетевым контуром проекта: включить в единый вероятностный граф не только сооружение блока, но и готовность схемы выдачи мощности, ОРУ/ЗРУ, внешних ЛЭП, релейной защиты, диспетчерских согласований и готовности энергосистемы страны к приёму мощности.
Обоснование (какую проблему решает, что это даёт): Решает риск ситуации, когда блок физически готов к ключевым этапам, но внешняя сеть или регуляторно-диспетчерский контур не готовы. Даёт Росатому возможность оптимизировать не только строительный график, но и весь путь до реально достижимой синхронизации/выдачи мощности, что особенно критично на экспортных проектах.

ID: R2-2-02
Идея: Применить модель как инструмент управления «социальной лицензией на строительство»: встроить в событийный граф общественные слушания, ОВОС/EIA, земельные вопросы, взаимодействие с местными властями, риски протестной активности и судебных/административных задержек.
Обоснование (какую проблему решает, что это даёт): Решает недооценённый, но реальный источник срыва сроков подготовительного периода, когда формально проект готов, но общественно-политический контур неустойчив. Даёт возможность сравнивать стратегии коммуникации, компенсационных мероприятий и фазирования работ по критерию «срок–стоимость–риск блокировки».

ID: R2-2-03
Идея: Применить модель для управления очередью нерешённых технических вопросов (TQ/RFI/waiver): каждое открытое техническое решение, междисциплинарное несоответствие или запрос к проектировщику/поставщику рассматривать как узел графа с downstream-влиянием на РД, закупки и фронты СМР.
Обоснование (какую проблему решает, что это даёт): Решает типовую проблему крупных АЭС, когда график срывают не «большие» события, а сотни мелких не закрытых вовремя технических вопросов. Даёт механизм приоритизации закрытия решений по их вероятностному влиянию на критический путь, а не по формальной очереди документооборота.

ID: R2-2-04
Идея: Применить модель как иерархический «сшиватель» графиков: связать единый вероятностный граф уровней 1–3 с тематическим планированием на 3 года и с 6-месячными графиками 4 уровня/частными потоками, чтобы верхнеуровневый прогноз автоматически раскладывался в исполнимые rolling-wave планы.
Обоснование (какую проблему решает, что это даёт): Решает разрыв между стратегическим графиком и реальным оперативным управлением площадкой. Даёт системный механизм, при котором пересчёт на верхнем уровне сразу меняет приоритеты в потоках, месячных окнах и краткосрочных пакетах, без потери связности модели.

ID: R2-2-05
Идея: Применить модель для оптимизации контура физзащиты и пропускной способности КПП как производственного ограничения: включить в расчёт доступ персонала, строительной техники, досмотров, сменных пиков, режимов доступа по зонам и этапность развёртывания защищённых контуров.
Обоснование (какую проблему решает, что это даёт): Решает скрытую потерю производительности, когда стройка теряет человеко-часы не на СМР, а на проходе, въезде, досмотре и пересечении режимных границ. Даёт количественное обоснование числа КПП, их мощности, очередности ввода и режимов доступа для пиковых фаз стройки, а не только формальное выполнение требований безопасности.

ID: R2-2-06

Механизм: моделирование пропускной способности КПП как ограничения; прогнозирование времени на процедуры доступа; оптимизация графика завоза материалов с учётом режимных ограничений; планирование поэтапного ужесточения режима; GERT-сценарии для периодов усиленного режима. Учитывает специфику режимного объекта в планировании.

Механизм: прогнозирование объёмов отходов по этапам строительства на основе событийно-временного графа; оптимизация логистики вывоза; учёт ограничений полигонов как ресурсного ограничения; GERT для сценариев с задержками вывоза; минимизация затрат при соблюдении экологических требований. Аспект, влияющий на логистику площадки.

**ID: R2-8-14**

**Обоснование:** Недостаточные или некачественные инженерные изыскания приводят к неожиданностям при строительстве (непредвиденные грунтовые условия, подземные воды). Механизм: вероятностная оценка полноты изысканий как входного параметра модели; связь качества изысканий с дисперсией длительностей земляных и фундаментных работ; ML-модель для оценки достаточности изысканий на основе характеристик площадки; многокритериальная оптимизация объёма дополнительных изысканий vs риски строительства. Учитывает фактор информационной неопределённости.

Механизм: GERT-модель процесса согласования ДС с вероятностями итераций; прогнозирование времени согласования по типам изменений и контрагентам; оптимизация порядка инициирования ДС; ML-классификация ДС по сложности согласования; учёт в общем графике как фактора неопределённости. Контрактный аспект, влияющий на сроки.

Механизм: построение графа внедрения ИС, синхронизированного с общим графиком; вероятностное моделирование длительности настройки и тестирования; GERT для сценариев доработки ПО; оптимизация очерёдности ввода систем по зависимостям ПНР.
