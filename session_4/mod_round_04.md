## 9. Сквозные паттерны доработок

Несколько конструктивных решений повторяются у разных участников, формируя консенсус:

**1. Безопасность = жёсткое ограничение, не переменная оптимизации**
Участники 1, 2, 7, 8 (и поддержка от У5, У6): Q выводится из F(Γ) = [T, C, R, Q] и переводится в constraint Q ≥ Q_min. Оптимизация — только внутри безопасного множества.

**2. Разделение DAG на управляемые и неуправляемые дуги**
Участники 1, 3, 8: внешние события (регулятор, политика, макроэкономика) — наблюдаемые ограничения среды; оптимизация только внутренних работ при разных сценариях внешних.

**3. Двухуровневая/двухгоризонтная архитектура**
Участники 1, 7, 8: стратегический пересчёт (месяц/квартал) vs оперативный мониторинг (неделя) с порогами значимости для предотвращения schedule jitter.

**4. ML → байесовские сети при малой выборке**
Участники 6, 8: для уникального оборудования (ОДЦИ) с n << 100 — переход от XGBoost/CatBoost к байесовским сетям, совмещающим экспертные оценки со статистикой.

**5. Телескопическая детализация**
Участники 1, 7, 8: укрупнённые узлы основного периода (уровень 1–2) разворачиваются по мере конкретизации; портфельный уровень работает на агрегированных P50/P80.

**6. Сужение области применения «маргинальных» идей**
Участники 5, 6: идеи, критикуемые за несоразмерность (вывоз грунта, энергопотребление, такелаж), ограничены конкретными условиями или отозваны.
