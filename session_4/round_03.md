## Раунд 3 — Критика

**C-1-01**
ID идеи: R1-3-01
Слабое место: Идея предлагает «единую цифровую платформу» для сквозного прогнозирования от подготовительного периода до физического пуска с обновлением «в реальном времени». Однако масштаб модели (тысячи работ в DAG обоих периодов, десятки тысяч дуг с вероятностными распределениями, ML-компонент для каждого класса операций) создаёт проблему вычислительной трактабельности: пересчёт полного Монте-Карло (10⁴–10⁵ итераций) по всему графу с одновременным обновлением ML-моделей и итеративной оптимизацией критических путей для сети из 5000+ работ может занимать часы-дни, а не «реальное время». Идея не конкретизирует ни уровень декомпозиции (1-й, 2-й, 3-й уровень графика), ни стратегию вычислений (инкрементальный пересчёт, иерархическая декомпозиция). Без этого заявление об обновлении «в реальном времени» остаётся неподтверждённым допущением, а не инженерным решением.

**C-1-02**
ID идеи: R1-4-05
Слабое место: Идея предлагает связать многомерный предиктивный граф из статьи с BIM/3D-моделью. Однако в статье Дмитришина пространство событий x_i = (t_i, ā_i) является абстрактным математическим пространством для оптимизации, а BIM оперирует геометрической 3D-моделью с семантикой элементов конструкций (IFC-классы). Между этими двумя представлениями нет прямого отображения: узел графа «бетонирование плиты Р2» из PERT-модели и IFC-объект «IfcSlab» в BIM — сущности из принципиально разных онтологий. Идея не определяет, каким образом устанавливается маппинг между событиями вероятностного графа и элементами BIM-модели, не обсуждает стоимость создания и поддержания этого маппинга для тысяч элементов, а заявленный результат («визуально видно, что готовность фундамента 80%») может быть достигнут значительно более простыми средствами (4D-визуализация в стандартных TICM-системах).

**C-1-03**
ID идеи: R1-5-10
Слабое место: Идея предлагает применить модель для управления финансовыми рисками (валютные колебания, инфляция) через многомерное пространство оптимизации. Но финансовые переменные (курсы валют, ставки, инфляция) определяются макроэкономическими факторами, полностью внешними по отношению к проекту и его сетевому графу. Модель Дмитришина оптимизирует траектории внутри DAG проекта, где дуги — это работы с управляемыми параметрами (сроки, ресурсы). Валютные колебания не являются дугой в графе проекта, не управляются через перемещение по DAG и не поддаются локальной оптимизации сегментов пути. Включение макроэкономических переменных как атрибутов ā_i в вектор состояния математически возможно, но оптимизация по ним бессмысленна — проект не может «выбрать траекторию» с лучшим курсом валюты. Идея подменяет предмет модели, распространяя её на область, где она не даёт управляющего воздействия.

**C-1-04**
ID идеи: R1-2-12
Слабое место: Идея предлагает использовать модель как «двигатель серийности» — переносить эмпирические распределения длительностей и типовые ветвления с проектов-аналогов на новые. Скрытое допущение: условия между проектами достаточно сопоставимы для прямого переноса статистических распределений. Однако каждый проект Росатома реализуется в уникальных условиях: другая площадка (климат, геология, логистика), другая страна-заказчик (для экспорта), другие подрядчики и поставщики, другие регуляторные требования. Эмпирические распределения длительностей T_ij, обученные на Ростовской АЭС, могут быть нерепрезентативны для АЭС «Аккую» или «Эль-Дабаа». Даже внутри России условия Нововоронежа и Курска существенно различаются. Без формализованной процедуры оценки переносимости (domain adaptation) и коррекции распределений под местные условия перенос может давать систематически смещённые прогнозы, создавая иллюзию точности.

**C-1-05**
ID идеи: R1-8-01
Слабое место: Идея формулирует максимально широко — «применение комплексной модели к управлению подготовительным периодом» — фактически повторяя постановку задачи всего мозгового штурма, а не предлагая конкретный механизм решения конкретной проблемы. Обоснование оперирует категориями «снижение рисков на 20–30%», «точность >85%» без указания, откуда взяты эти цифры и к какому показателю они относятся. Идея рискует остаться декларативным зонтичным предложением, которое невозможно реализовать как единое инженерное решение и которое при попытке конкретизации распадётся на те же отдельные приложения, которые уже предложены другими участниками в более конкретной форме.

**C-1-06**
ID идеи: R2-5-12
Слабое место: Идея предлагает применить модель для оптимизации графика утилизации строительных отходов и вывоза грунта. Хотя логистика отходов действительно является практической проблемой, масштаб её влияния на критический путь АЭС несоразмерен сложности применяемого аппарата. Вывоз грунта — логистическая задача, решаемая стандартными методами оперативного планирования (диспетчеризация транспорта, ёмкости полигонов). Применение полного комплекса вероятностных PERT/GERT-сетей, ML-прогнозирования и многокритериальной оптимизации к задаче вывоза отходов представляется избыточным и не соответствует задаче мозгового штурма — искать такие применения модели, которые создают значимую бизнес-ценность при сооружении АЭС. Трудозатраты на построение и калибровку модели для этой подзадачи не оправдаются её влиянием на ключевые параметры проекта (T, C, R).

**C-1-07**
ID идеи: R1-1-14
Слабое место: Собственная идея об обучении ML-моделей на данных серийного строительства содержит скрытую проблему: для формирования полноценной обучающей выборки D = {(x_ij^(k), y^(k))} с достаточной статистической мощностью необходимы данные десятков завершённых блоков по каждому типу операций. У Росатома на данный момент количество завершённых блоков поколения III+ (ВВЭР-1200) ограничено несколькими единицами (Нововоронежская-2, Ленинградская-2, части зарубежных), причём их данные могут быть собраны в разных форматах и с разной степенью детализации. Для ML-методов, требующих n ≥ 30 наблюдений (порог перехода к эмпирическим распределениям в статье), это ограничение является критическим для многих типов операций. Модель рискует быть переобученной на малой выборке и давать ложную уверенность в прогнозах.

**C-1-08**
ID идеи: R2-6-03
Слабое место: Идея предлагает оптимизацию принятия решений при такелаже сверхтяжёлого оборудования методом Open Top с использованием деревьев решений и критерия Вальда. Однако такелажная операция «подъём корпуса реактора» — это единичное событие за весь срок строительства блока, длящееся несколько часов. Решение «поднимать / не поднимать» принимается командой такелажа на основе текущих метеоданных и допусков крановой техники в реальном времени. Построение полноценной вероятностной GERT-модели с ML-компонентом для события, происходящего один раз и решаемого за минуты оперативным персоналом, представляется несоразмерным: трудозатраты на разработку, калибровку и валидацию модели значительно превысят ценность формализации решения, которое и без модели принимается компетентно инженерами-такелажниками по стандартным процедурам.

**C-1-09**
ID идеи: R1-7-14
Слабое место: Идея предлагает оптимизацию правовых, контрактных и бизнес-решений (формирование СП, распределение прибыли, трансфертное ценообразование) через вероятностную модель. Правовые и бизнес-решения определяются переговорной позицией сторон, юридическими рамками, политическими факторами, а не оптимизацией сетевого графа работ. Модель Дмитришина оптимизирует траектории выполнения работ в DAG, а не распределение прибыли между юридическими лицами. Попытка формализовать переговорный процесс через PERT/GERT-сеть подменяет юридическую и бизнес-логику инженерной оптимизацией и может дать формально «оптимальные», но юридически ничтожные или политически неприемлемые решения.

**C-1-10**
ID идеи: R2-5-11
Слабое место: Идея предлагает интеграцию модели с системой мониторинга энергопотребления строительной площадки. Электроснабжение строительной площадки планируется на этапе проектирования временного энергоснабжения с известными установленными мощностями потребителей (краны, сварочные посты, БРХ). Пиковые нагрузки определяются составом одновременно работающего оборудования и предсказуемы по графику работ без применения вероятностных методов. Ситуация «ограничение по электроснабжению блокирует работы» на крупной стройке АЭС при нормальном проектировании не должна возникать — мощность временного электроснабжения проектируется с запасом. Это маргинальная проблема, не оправдывающая развёртывание комплексной вероятностной модели.

**C-1-11**
ID идеи: R1-4-09
Слабое место: Идея предлагает использовать Монте-Карло для оценки достаточности объёма инвестиций в подготовительный период, включая сценарии «остановка проекта на разных этапах». Однако решение об остановке проекта АЭС — это стратегическое политическое решение государственного уровня, определяемое энергетической политикой, межгосударственными соглашениями и обязательствами (для экспортных проектов). Вероятность остановки проекта не может быть корректно оценена статистическими методами на основе исторических данных — слишком мало наблюдений, слишком уникальны причины. Присвоение конкретной вероятности событию «проект будет остановлен на этапе X» создаёт иллюзию квантификации принципиально неквантифицируемого политического риска.

**C-1-12**
ID идеи: R1-1-11
Слабое место: Собственная идея о расширении модели на портфельный уровень предполагает построение связанных PERT/GERT-сетей для нескольких проектов одновременно с общими ресурсными ограничениями. Практическая проблема: графики проектов Росатома в разных странах находятся под разными контрактными режимами, разными системами управления, разным уровнем детализации и имеют различную степень коммерческой конфиденциальности. Объединение графиков разных юрисдикций в единую оптимизационную модель наталкивается не на математические, а на организационные и правовые барьеры — разные дивизионы Росатома, разные EPC-контракторы, разные системы отчётности. Размерность задачи портфельной оптимизации (десятки тысяч работ × десятки проектов × общие ресурсы) может оказаться за пределами вычислительной трактабельности для методов, описанных в статье.

**C-1-13**
ID идеи: R2-3-05
Слабое место: Идея предлагает адаптивный выбор критерия принятия решений (Вальд → Байес → Гурвиц) в зависимости от фазы проекта и зрелости данных. Сам по себе подход разумен, но слабое место — критерий перехода: кто и на каком основании определяет, что модель «перешла» из режима высокой неопределённости (Вальд) в режим достаточной статистики (Байес)? Если переход определяется формальным порогом (например, n ≥ 30 наблюдений), возникает дискретный скачок в характере принимаемых решений при переходе через порог, что может приводить к нестабильности рекомендаций. Если переход определяется экспертно — теряется заявленная «адаптивность». Идея не специфицирует этот механизм, оставляя ключевой вопрос нерешённым.

**C-1-14**
ID идеи: R1-2-03
Слабое место: Идея предлагает использовать модель как систему увязки проекта-аналога, площадочной адаптации и управления требованиями — «от выбора референтной АЭС до прослеживания того, как каждое требование влияет на проектирование, закупки, график и риски». Управление требованиями (requirements management) — это отдельная дисциплина со своими стандартами (ISO 15288, INCOSE), инструментами (DOORS, Polarion) и методологией (трассировка, верификация, валидация). Модель Дмитришина — это инструмент оптимизации и прогнозирования графика, а не система управления требованиями. Попытка превратить вероятностный сетевой граф в систему прослеживания требований размывает фокус модели и конкурирует с уже существующими специализированными инструментами, для которых формализация «как каждое требование влияет на проектирование» — основная функция. Модель может использоваться для оценки влияния конкретного изменения требования на график (что ближе к идеям из К21), но не для полноценного управления требованиями.

**C-1-15**
ID идеи: R2-7-02
Слабое место: Идея предлагает интеграцию показателей «культуры безопасности» и «человеческого фактора» в событийно-временной граф. Проблема в измеримости: «культура безопасности» — это социально-психологический конструкт, не имеющий общепринятой числовой метрики. Предлагаемые суррогатные атрибуты («уровень переработок, доля неопытного персонала, доля субподрядчиков, история замечаний по качеству») — это коррелирующие, но не причинные факторы. ML-модель, обученная на этих косвенных признаках, будет прогнозировать не вероятность инцидента как таковую, а корреляцию с условиями, в которых инциденты наблюдались ранее. При малом количестве реальных инцидентов на стройках АЭС (к счастью) обучающая выборка для событий «инцидент → расследование → переделка» будет слишком мала для надёжного ML-прогноза, что делает модель уязвимой к переобучению и ложным корреляциям.

Комментарий участника: Критика направлена на выявление конкретных слабых мест по существу — скрытых допущений, проблем масштабирования, ограничений применимости модели и несоразмерности аппарата задаче. Часть критик адресована собственным идеям (C-1-07, C-1-12), где выявлены реальные ограничения.

Есть риск переоценить возможность «оптимизировать» погоду и получить лишь перераспределение задержек по календарю, а не их реальное устранение.

ID идеи: R2-7-06
Слабое место: Автоматический пересчет сетевой модели на основе телеметрии (GPS кранов, фотограмметрия, БПЛА) несет критический риск искажения статуса проекта. На ОИАЭ физическое возведение конструктива не означает завершения работы. Работа считается завершенной только после прохождения лабораторного контроля, оформления исполнительной документации и подписания актов (зачастую с участием Ростехнадзора или инозаказчика). Телеметрия переведет работу в статус «выполнено» преждевременно, что заставит оптимизатор выдавать ложные команды на открытие смежных фронтов до их легитимной документальной приемки.

**C-6-05**
ID идеи: R2-8-05
Слабое место: Математическая нормализация «расползания объема» (scope creep) как ожидаемого стохастического процесса, под который алгоритмически закладываются резервы, разрушает административную дисциплину «заморозки проекта» (Design Freeze). Если модель заранее легитимизирует появление неучтенных изменений и алгоритмически резервирует под них ресурсы, проектировщики и заказчик с высокой вероятностью выберут этот лимит (согласно закону Паркинсона). Это алгоритмически поощряет порочную практику выпуска сырой документации и потери контроля над бюджетом.

**C-6-06**
ID идеи: R1-3-06
Слабое место: Непрерывное обучение ML-модели на данных «всех текущих и завершенных проектов Росатома» (в РФ и за рубежом) игнорирует фундаментальную гетерогенность данных. Экспортные проекты реализуются в совершенно иных климатических условиях, с другой производительностью местного труда, иной логистикой и под надзором других национальных регуляторов. ML-модель, обученная на «смеси» таких разнородных данных, будет страдать от смещения (data shift) и выдавать некорректные, усредненные прогнозы, неприменимые к специфическому страновому контракту.

**C-6-07**
ID идеи: R2-2-05
Слабое место: Массовые бетонные работы (включая первый бетон фундаментной плиты) жестко регламентированы технологическими картами, правилами (ПиН АЭ) и СНиП. В них прописаны строгие требования к непрерывности укладки, температурному контролю (изотермия) и времени твердения. Пространство для «алгоритмической оптимизации» самого процесса бетонирования практически отсутствует — невозможно математически изменить скорость гидратации цемента или безопасно «поставить на паузу» заливку без риска образования холодных швов. Модель пытается оптимизировать технологически ригидный процесс.

**C-6-08**
ID идеи: R2-1-03
Слабое место: Идея моделирования конвейера подготовки персонала (сварщиков, дефектоскопистов) исходит из допущения, что генподрядчик обладает полными данными о внутренних HR-процессах. На практике на монтаж привлекаются специализированные субподрядные организации со своим штатом. Генподрядчик не имеет прямого доступа к данным об их внутренних циклах обучения, процентах сдачи экзаменов в учебных центрах и воронке найма. Без этих первичных данных GERT-сеть и ML-прогнозирование кадрового конвейера останутся лишь теоретической абстракцией без фактического наполнения.

**C-6-09**
ID идеи: R2-3-06
Слабое место: Юридическая неприменимость стохастического контрфактического анализа в претензионной работе. Инженерно-юридические практики разрешения строительных споров (например, SCL Delay and Disruption Protocol) и международные арбитражи требуют детерминированного доказательства влияния конкретного события на критический путь (методы Time Impact Analysis, As-Planned vs As-Built). Вероятностная модель Монте-Карло, утверждающая, что «без этого события с вероятностью 80% проект завершился бы раньше», не будет принята заказчиком или арбитражем в качестве легитимного доказательства для защиты от многомиллионных штрафов (Liquidated Damages), так как не устанавливает однозначную де-факто причинно-следственную связь.

Комментарий участника: Анализ направлен на выявление конфликтов предложенных математических моделей с жесткими физическими, технологическими, математическими, правовыми и нормативными (регуляторными) ограничениями предметной области сооружения АЭС.

## Раунд 3 — Критика

**C-7-01**
ID идеи: R1-3-02
Слабое место: Идея предполагает наличие хорошо формализованных и статистически описанных процессов лицензирования, разработки ПОС, контрактации и подготовки площадки, тогда как для многих экспортных проектов такие данные единичны, нерепрезентативны и сильно зависят от политического контекста. В результате вероятностные распределения по ключевым регуляторным и контрактным событиям будут строиться в основном на экспертных оценках, что даёт иллюзию точности, но плохо валидируется. Кроме того, единая крупная сеть на весь подготовительный период рискует стать слишком громоздкой и непрозрачной для практического применения: управленцам будет трудно понять, какие конкретно решения нужно менять по результатам оптимизации.

**C-7-02**
ID идеи: R1-3-03
Слабое место: Для работы динамического управления тепломонтажом и электромонтажом с недельным горизонтом нужны подробные и достоверные фактические данные по прогрессу работ (на уровне узлов графа), которых сейчас часто нет или они сильно запаздывают. В таких условиях ML-модели будут обучаться на шумных и неполных данных, а пересчёт критического пути может давать противоречивые рекомендации. Дополнительно, частая переоптимизация графика может конфликтовать с жёсткими контрактными схемами и технологиями производства работ: подрядчики не готовы каждую неделю менять фронты и приоритеты, поэтому риск управленческого «перегрева» и недоверия к системе высок.

**C-7-03**
ID идеи: R1-5-05
Слабое место: Идея исходит из того, что влияние дополнительных вложений в подготовительный период на снижение рисков основного периода можно количественно оценить и заложить в F(Γ) = [T, C, R, Q], но для многих решений (глубина проработки, качество исходных данных, степень опережающей контрактации) причинно-следственная связь с реальными задержками в основном периоде слабо формализована. Если эти зависимости описать слишком грубо, оптимизация даст «красивый» Парето-фронт, который не будет отражать реальные узкие места. Есть риск, что модель предложит экономически оптимальные, но организационно или регуляторно невыполнимые траектории (например, уровень готовности площадки, несовместимый с ограничениями по финансированию или земельным процедурам).

**C-7-04**
ID идеи: R1-5-08
Слабое место: Интеграция событийно-временного графа с BIM/цифровым двойником опирается на очень высокий уровень зрелости цифрового моделирования: актуальность моделей, полная трассировка объектов, единая система идентификаторов между BIM и графиком. На многих площадках BIM используется фрагментарно и не всегда поддерживается в состоянии «as built», поэтому автоматическое сопоставление событий графика и состояния помещений может оказаться технически недостижимым или потребовать непропорционально больших усилий по поддержанию данных. В итоге есть риск получить сложную интеграцию, которая работает только на ограниченном наборе объектов и не даёт системного эффекта по всей АЭС.

**C-7-05**
ID идеи: R2-2-01
Слабое место: Для построения PERT/GERT-контура по мобилизации, обучению, аттестации и удержанию персонала с учётом СБИ нужны детализированные данные по производительности бригад, текучести, влиянию жилья, питания и медицины — такие данные в реальных проектах либо не собираются системно, либо разрозненны у разных подрядчиков и часто закрыты. Без надёжной статистики ML-компонент по факту останется набором экспертных коэффициентов, а модель не сможет корректно прогнозировать «критичность» персонала. Кроме того, значительная часть решений по СБИ и кадровой политике принимается на более высоком корпоративном/государственном уровне, чем уровень управления стройкой; оптимальный по модели путь может оказаться управленчески нереализуемым для конкретного проекта.

**C-7-06**
ID идеи: R2-3-01
Слабое место: Совмещение сетевой модели строительства с детализированным вероятностным cash-flow и оптимизацией NPV/IRR создаёт очень тяжёлый по размерности контур, где смешиваются задачи строительного управления и корпоративных финансов. В реальности финансовая модель проекта часто живёт отдельно, в другой системе и под ответственностью другой команды; синхронизация двух контуров в режиме, достаточном для регулярной оптимизации, может быть организационно невыполнимой. Кроме того, входные параметры финансового блока (ставки, курсы, условия кредитов) сильно волатильны, и при их изменении придётся полностью пересчитывать оптимальное решение — есть риск, что модель станет слишком сложной и не будет использоваться операционно, оставаясь «аналитическим упражнением».

**C-7-07**
ID идеи: R2-1-05
Слабое место: Идея опирается на то, что показатели EVM (SPI, CPI и др.) рассчитаны качественно и на сопоставимой основе по всему проекту, но на практике освоенный объём часто формируется по укрупнённым, частично субъективным оценкам и сильно зависит от методики конкретного проекта. Попытка встроить такие показатели в вероятностную сетевую модель и поверх этого строить ML-прогнозы EAC/ETC может привести к усилению артефактов исходных данных, а не к более точной оценке. Дополнительно, интеграция EVM с событийно-временным графом требует жёсткого выравнивания кодов работ и структур WBS, что в текущих процессах Росатома далеко не всегда выдержано — трудоёмкость наведения такой дисциплины может превысить выгоды от самой интеграции.

**C-7-08**
ID идеи: R2-7-06
Слабое место: Связка модели с цифровым двойником стройки через BIM, СКУД, телеметрию кранов, фотограмметрию и журналы качества предполагает, что все эти системы внедрены, стандартизованы и регулярно пополняются данными, чего сейчас зачастую нет даже на пилотных площадках. Без устойчивого контура сбора и очистки данных автоматический пересчёт сетевой модели по факту может быть либо невозможен, либо будет опираться на неполные датасеты и приводить к «ложным тревогам». Кроме того, юридически и контрактно прогресс работ всё равно фиксируется актами и отчётностью, а не данными цифрового двойника; если управленческие решения и платежи не завязаны на автоматическую оценку готовности, ценность такого модуля для менеджмента останется ограниченной.

Комментарий участника: Без комментария.

Формирование атрибута «потребление энергии» в каждой вершине вектора xi требует значительной инженерной работы по нормированию, которая непропорциональна производственному эффекту от оптимизации.

**C-8-05**

Применение полного аппарата модели Дмитришина — PERT+GERT+Монте-Карло+ML+оптимизация — для этой задачи представляет избыточность: задача решается существенно более простыми инструментами (оперативное ресурсное планирование). Идея не обосновывает, почему именно комплексная вероятностная модель является необходимым и соразмерным инструментом для данной конкретной задачи.

**C-8-06**

Однако в реальности проекты Росатома реализуются в рамках EPC-контрактов с зафиксированными обязательствами: генподрядчик, мобилизованный на конкретный проект, не может быть «переключён» на другой без нарушения контрактных обязательств, штрафов и демобилизационно-мобилизационных издержек. Краны и персонал в большинстве случаев принадлежат субподрядчикам, а не Росатому напрямую. Оптимизационный результат модели на портфельном уровне может быть технически корректным, но практически нереализуемым в условиях действующих контрактных структур.

**C-8-07**
ID идеи: R1-8-19 / R2-3-08 (К23 — стресс-тестирование)
Слабое место: Идеи стресс-тестирования предполагают, что сценарии внешних шоков (санкции, остановка генподрядчика, отзыв лицензии) поддаются параметризации в виде вероятностных состояний природы sj с известными или оцениваемыми вероятностями pj. Однако события типа «введение санкций» или «политический форс-мажор в стране-заказчике» — это события с принципиально неизвестными вероятностями (неопределённость по Найту, а не риск). Применение критерия Вальда (максиминная стратегия) для «чёрных лебедей» даёт тривиальный результат — максимально консервативный план, — но не помогает принимать управленческие решения в реальных условиях неизмеримой неопределённости. Модель дисциплинирует мышление, но не решает проблему параметризации самих сценариев.

**C-8-08**
ID идеи: R1-6-01 / R1-5-02 / R1-8-07 (К3 — управление поставками ОДЦИ)
Слабое место: Прогнозирование рисков поставок ОДЦИ с применением ML требует обучающей выборки по историческим задержкам поставщиков. Однако поставщики ОДЦИ для Росатома (завод «Атоммаш», КМПО, «Звезда» и другие) — это единичный или очень ограниченный перечень производителей, каждый из которых обслуживает несколько проектов параллельно. Исторических наблюдений по задержкам конкретного поставщика конкретного типа оборудования — недостаточно для статистически значимого обучения ML-модели (десятки наблюдений, а не тысячи). XGBoost, Random Forest, CatBoost работают значительно лучше при n >> 100 наблюдений. На малых выборках более надёжна байесовская калибровка экспертных оценок, а не статистическое машинное обучение. Эта проблема малой выборки не обсуждается ни в одной из идей.

**C-8-09**
ID идеи: R2-6-01 (К12 — сглаживание эпюры финансирования)
Слабое место: Идея предлагает алгоритмически сдвигать события на субкритических путях для сглаживания кассовых разрывов. Однако в контрактах генподряда АЭС платёжные условия, как правило, жёстко привязаны к контрактным вехам (milestone payments), которые согласованы между заказчиком и генподрядчиком. Сдвиг события на субкритическом пути означает изменение контрактной вехи, что требует двустороннего согласования и оформления дополнительного соглашения. Алгоритм не может самостоятельно изменять контрактные обязательства, поэтому «свобода манёвра» на субкритических путях ограничена контрактной архитектурой проекта, что не отражено в идее.

**C-8-10**
ID идеи: R1-8-16 / R1-1-07 (К6 — GNN-модели для прогнозирования длительностей)
Слабое место: GNN-модели (GCN, GAT, Temporal Graph Networks) требуют для обучения большого числа структурно однородных графов с известными метками (фактические длительности). График АЭС — это уникальный крупномасштабный DAG, каждый проект которого имеет принципиально иную конфигурацию по сравнению с другими проектами. Попытка обучить GNN на нескольких (5–15) проектах Росатома при их структурном разнообразии даст модель с высокой дисперсией. Temporal Graph Networks, строго говоря, предназначены для динамических графов (социальные сети, транзакционные данные), где тысячи однотипных рёбер появляются и исчезают в реальном времени — это принципиально иная структура данных, чем статический DAG строительного проекта. Применимость GNN к данной задаче требует отдельного обоснования.

**C-8-11**
ID идеи: R2-8-03 (К19 — прогноз рисков травматизма и ядерной безопасности)
Слабое место: Прогнозирование вероятности производственного инцидента для конкретного временного интервала на основе «количества одновременно выполняемых работ» и «накопленных сверхурочных часов» является технически обоснованным подходом для обычного промышленного строительства. Однако сооружение объектов ядерного цикла регулируется культурой безопасности, которая требует не оптимизации расписания работ ради снижения статистического риска, а полного выполнения требований ОПБ, НП и инструкций по безопасности независимо от экономической стоимости. Перераспределение нагрузки «для минимизации вероятности инцидента» на основе ML-прогноза без учёта регуляторных требований создаёт риск подмены нормативно-обязательных мер безопасности вероятностной оптимизацией, что неприемлемо в ядерной отрасли.

Исторические данные по срокам лицензирования немногочисленны и нерепрезентативны: каждый лицензионный процесс уникален по набору условий. Оценка вероятности «отказа в выдаче лицензии» с любой разумной точностью методами ML или экспертными методами является крайне ненадёжной. Модель создаёт иллюзию управляемости процесса, который по природе своей слабо формализуем.

**C-8-13**

Однако в строительстве АЭС персонал с атомными допусками — это дефицитный рынок: сварщики с НАКС, НК-специалисты, пусконаладчики нанимаются через субподрядные организации, а не через прямое управление HR-конвейером. Оптимизация момента начала аттестации предполагает, что Росатом или генподрядчик контролируют кадровый поток. На практике же субподрядчики нанимают, обучают и аттестуют персонал самостоятельно, и прямое управление их HR-процессами со стороны заказчика невозможно. Модель может прогнозировать потребность, но не управлять конвейером на стороне субподрядчика.

**C-8-14**
ID идеи: R1-8-21 (К7 — обоснование концепции сооружения в рамках стандарта)
Слабое место: Идея предполагает, что альтернативные концепции сооружения (разные проекты-аналоги, схемы водоснабжения, состав СМБ) могут быть сравнены через Парето-фронт по критериям T, C, R, Q на предпроектной стадии. Квантитативное сравнение при этом создаёт ложную точность и может использоваться для постфактического обоснования уже принятого решения, а не для его подлинно аналитического выбора.
