## 4. Основные паттерны ответов

### 4.1. Модель → инструмент поддержки решений (DSS), не автопилот
Большинство участников (1, 2, 3, 6, 7, 8) в ответах перевели свои идеи из режима «автоматическая оптимизация» в режим «подготовка обоснования + рекомендации для ЛПР». Это сквозная тенденция.

### 4.2. Внешние ограничения → жёсткие граничные условия
Участники 1, 2, 5, 7 систематически переводят неконтролируемые факторы (регулятор, политика, рынок) из оптимизируемых переменных в жёсткие ограничения модели. Модель оптимизирует только внутри зоны контроля Росатома.

### 4.3. ML → гибридные схемы с экспертным контролем
Участники 3, 4, 6, 8 предложили переход от чистого ML к гибридным подходам: ML для прескрининга и приоритизации, детерминированные методы и экспертное суждение для финальных решений.

### 4.4. Дефицит данных → адаптивный подход
Участники 1, 3, 5 предложили механизм «fallback»: при недостатке данных модель автоматически расширяет доверительные интервалы, переключается на консервативные критерии (Вальд) и выдаёт метрику уверенности.
