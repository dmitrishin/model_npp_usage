## 4. Основные тематические линии критики

### 4.1. Дефицит данных для обучения ML/GNN
Критики: C-1-01, C-1-10, C-3-02, C-3-03, C-3-08, C-4-01, C-4-03, C-4-06, C-5-01, C-5-06, C-6-10  
Суть: количество завершённых проектов АЭС крайне мало для обучения ML/GNN; данные гетерогенны, фрагментарны, часто закрыты по режимным требованиям.

### 4.2. Разрыв между математическим оптимумом и организационной реальностью
Критики: C-1-02, C-1-09, C-5-08, C-6-01, C-6-04, C-7-04, C-7-06  
Суть: автоматическая генерация решений не учитывает контрактные, юридические, организационные ограничения; переброска ресурсов требует времени на допуски, инструктажи, демобилизацию.

### 4.3. Неконтролируемые внешние факторы
Критики: C-1-11, C-2-03, C-2-08, C-3-04, C-3-09, C-4-05, C-5-04, C-7-05, C-8-03  
Суть: политические решения, поведение регулятора, рынок страхования, валютные факторы не являются управляемыми параметрами модели.

### 4.4. Несоответствие механизма модели предметной области
Критики: C-1-05, C-1-12, C-1-13, C-1-16, C-1-17, C-3-10, C-6-07  
Суть: ряд задач (углеродный след, теплофизика хранения топлива, земельные процедуры, переклассификация оборудования) не соответствует механизмам модели Дмитришина.

### 4.5. NLP для ядерного регулирования
Критики: C-1-04, C-1-18, C-5-03, C-5-07, C-6-03, C-7-02  
Суть: NLP-модели не способны гарантировать точность при сравнении сложных ядерных норм; регуляторная переписка конфиденциальна и малочисленна.

### 4.6. RL в строительстве АЭС
Критики: C-1-09, C-2-14, C-4-09, C-5-02, C-6-02, C-7-03  
Суть: sim-to-real gap, невозможность определить функцию вознаграждения, отсутствие среды для обучения, непрозрачность решений.
