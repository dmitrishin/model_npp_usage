Место: 1
ID идеи: R1-1-04
Идея: Оптимизация критического пути основного периода, определяемого ТМР и сваркой трубопроводов — вероятностное моделирование длительностей сварочных операций, GERT-циклы «брак → ремонт → повторный НК», Monte Carlo для выявления критических и субкритических путей, итеративная оптимизация распределения бригад, ML-прогноз длительностей по характеристикам стыков.
Почему: Безусловный лидер по итогам всех раундов (73 балла в Раунде 5, голоса от 4 участников). ТМР и сварка — общепризнанный и документально подтверждённый критический путь строительства АЭС (54–65+ мес.). Модель применяется комплексно и по прямому назначению — все компоненты (PERT, GERT, Monte Carlo, локальная и итеративная оптимизация, ML) задействованы естественным образом. Максимальная бизнес-ценность: сокращение критического пути даже на 2–3 месяца даёт экономию в сотни миллионов рублей и ускоряет начало генерации электроэнергии. Критика в Раунде 3 была направлена на смежные идеи, сама идея устояла без существенных доработок.

Место: 2
ID идеи: R1-1-13
Идея: Количественная оценка готовности проекта к переходу от подготовительного к основному периоду — probabilistic go/no-go шлюз перед Первым бетоном. Множество предусловий ПБ как состояние в многомерном пространстве атрибутов, Monte Carlo для оценки вероятности достижения целевых сроков при различных уровнях готовности, критерии принятия решений для обоснования решения «идти / отложить / идти с компенсирующими мерами».
Почему: Решение о ПБ при неполной готовности — одна из самых дорогостоящих ошибок в проектах АЭС, каскадно влияющая на весь основной период. Критерии принятия решений из статьи (Вальд, Гурвиц) применяются строго по назначению — формализация управленческого gate. Модель превращает экспертное совещание в расчётную процедуру. Подкреплена близкими идеями R1-2-02, R1-3-14 — широкий консенсус участников. Предпочитаю R1-1-13 перед R1-2-02, так как R1-1-13 явно включает механизм количественной оценки влияния неполной готовности на основной период.

Место: 3
ID идеи: R1-1-02
Идея: Количественный анализ рисков графика и стоимости (QRA) при формировании концепции сооружения — Monte Carlo для распределений сроков и стоимости, GERT для сценарных развилок, критерии решений для стратегии формирования резервов.
Почему: QRA — стандартная международная практика (AACE, IAEA), прямо требуемая рекомендациями (H2). Модель Дмитришина — готовый математический аппарат для QRA. Результат — обоснованные базовые линии с количественно определёнными резервами вместо экспертных процентов. Критика касалась организационного принятия и зрелости данных — это ограничения внедрения, не обесценивающие механизм. Доработка (Раунд 4): вероятностная модель «под капотом», на выходе — детерминированные цифры с обоснованием для бюджетного процесса.

Место: 4
ID идеи: R2-1-02
Идея: Управление процессом контроля качества (несоответствия, переделки, ремонт) как отдельным вероятностным слоем графа — «граф качества», где каждая контрольная точка — GERT-узел с вероятностями исходов, ML-прогноз вероятности брака по типам работ/подрядчикам/условиям, оптимизация профилактических мер.
Почему: Новый механизм Раунда 2, получивший наивысшую поддержку среди новых идей (53 балла, 5 участников голосовали). Переделки — скрытый, но мощный фактор удлинения критического пути, который не виден в детерминированных графиках. Выделение качества в отдельный слой — практичный подход: статистика НК ведётся обязательно, данные доступны. Поддержан тремя независимыми участниками (R2-2-06, R2-3-02, R2-4-03). Проактивное управление качеством, ориентированное на график — прямая производственная ценность.

Место: 5
ID идеи: R1-1-01
Идея: Оптимизация сквозного графика подготовительного периода (от принятия решения до ПБ) с вероятностной оценкой достижения даты ПБ. Доработана: rolling-wave — укрупнённый граф (20–50 вех) на этапе 1 с широкими распределениями, детализация по мере продвижения, явная демонстрация диапазона неопределённости.
Почему: Подготовительный период (2–8 лет) — фундамент всего проекта, определяющий успех основного периода. Пять из шести участников предложили аналогичные идеи — максимальный консенсус. Rolling-wave доработка (Раунд 4) снимает критику о недостатке данных. Все компоненты модели работают комплексно. Количественная демонстрация неопределённости на ранних стадиях — сама по себе ценный управленческий результат.

Место: 6
ID идеи: R1-1-03
Идея: Оптимизация стратегии опережающей контрактации оборудования — определение оптимальных сроков каждой из шести волн контрактации с учётом неопределённости изготовления, готовности ИТТ/ЗЗИ, финансовых ограничений, GERT-ветвления квалификации поставщиков, многокритериальная оптимизация (стоимость хранения vs. риск неготовности).
Почему: Опережающая контрактация — один из ключевых факторов сокращения сроков основного периода, прямо зафиксированный в стандарте и рекомендациях (G5). Решение «когда запускать волну» — конкретная задача оптимизации, к которой модель применяется естественно. Многокритериальная оптимизация «just-in-time с учётом рисков» — прямой формализм статьи. Высокая бизнес-ценность: правильный тайминг контрактации сокращает основной период на месяцы.

Место: 7
ID идеи: R1-1-06
Идея: Планирование и оптимизация этапов ПНР (А-0 — Г-2) с учётом зависимости от хода строительства и монтажа — GERT-сеть испытаний с ветвлениями «успех / повторение», Monte Carlo для вероятности подэтапов в срок, интегрированный план «строительство–монтаж–ПНР». Доработана: contingency nodes для emergent work на головных блоках.
Почему: ПНР — этап с максимальной неопределённостью, где GERT-модель даёт наибольшую добавочную ценность по сравнению с детерминированным планированием. Итеративный характер испытаний (ХГО, испытания герметичности, физпуск) естественно описывается вероятностными ветвлениями. Интеграция с графиком строительства даёт сквозной прогноз до физпуска — то, чего нет в текущей практике. Доработка contingency nodes усиливает применимость для головных блоков.

Место: 8
ID идеи: R1-1-10
Идея: Оптимизация графика выпуска рабочей документации — сеть зависимостей комплектов РД как DAG, GERT для итераций согласования ИДП, Monte Carlo для вероятности готовности к требуемым датам, ML-прогноз длительности разработки, приоритизация комплектов по влиянию на критический путь.
Почему: Второе место по баллам в Раунде 5 (56 баллов, 4 участника). График выпуска РД — «один из самых ранних плановых документов, задающий темп всех последующих процессов» (G4). Задержка РД каскадно задерживает СМР и монтаж. Приоритизация комплектов по влиянию на критический путь — конкретный, практически применимый результат для проектной организации. Модель адресует реальную межорганизационную зависимость: технологическая, строительная, электротехническая дисциплины + поставщики.

Место: 9
ID идеи: R2-1-06
Идея: Оптимизация графика плановых регуляторных инспекций и проверок Ростехнадзора в увязке с ходом строительства — hold points как обязательные GERT-узлы, вероятность готовности к инспекции, длительность инспекции, вероятность замечаний и повторной инспекции, оптимальные даты подачи заявок.
Почему: Регуляторные hold points — реальные блокировки строительства, не учитываемые в стандартном сетевом планировании. Ожидание инспекции или неготовность к ней создаёт простои, измеряемые неделями. GERT-петли «инспекция → замечания → устранение → повторная инспекция» — прямое и естественное применение модели. Новый механизм Раунда 2, получивший голоса от 3 участников. Высокая практическая ценность: синхронизация строительства с регуляторным процессом снижает непроизводительные простои.

Место: 10
ID идеи: R1-1-14
Идея: Оптимизация графика поставок оборудования — полная цепочка «контрактация → производство → доставка → приёмка → монтаж» как GERT-сеть с ветвлениями (повторные инспекции, задержки транспортировки, несоответствие при входном контроле), Monte Carlo для распределения дат поступления, многокритериальная оптимизация «ранний заказ vs. риск несвоевременной поставки».
Почему: Поставки оборудования по шести волнам — системная задача, связывающая подготовительный и основной периоды. Интегрированный вероятностный график поставок — то, чего не даёт детерминированное планирование. GERT-ветвления для реальных сбоев (брак на заводе, задержка логистики, таможня) делают модель практически ценной. ML на данных фактических сроков изготовления аналогичного оборудования повышает точность. Особенно ценно для зарубежных проектов с длинными и сложными логистическими цепочками.
