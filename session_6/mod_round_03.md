### Основные тематические направления критики

1. **Недоступность и качество данных** — ML-модели, байесовские калибровки, финансовые данные подрядчиков требуют объёмов и качества данных, которых у Росатома пока нет
2. **Выход за пределы применимости модели** — кибербезопасность, ESG, экология, РАО, энергоэффективность, управление ИС — непрофильные для модели Дмитришина области
3. **Субъективность входных параметров** — вероятности состояний природы, полезности, экспертные оценки создают иллюзию объективности
4. **Организационные и политические барьеры** — автономия проектов, контрактные обязательства, роль регулятора, политические решения
5. **Вычислительная и интеграционная сложность** — масштаб моделей, синхронизация с BIM, цифровизация площадки, «проклятие размерности»
6. **Юридические ограничения** — признание вероятностных моделей в арбитраже, доказательная сила Monte Carlo

## Сводный перечень критических замечаний

### Участник 1 (16 замечаний)

**C-1-01** — ID идей: R1-1-17, R1-3-04, R1-4-14, R2-3-08  
Все идеи по обучению ML на данных реализованных проектов Росатома предполагают, что данные существуют в структурированном, полном и достоверном виде. На практике данные разрозненны, содержат систематические искажения, и стоимость ретроспективной очистки ни в одной идее не оценивается.

**C-1-02** — ID идеи: R2-3-04  
Теоретико-игровое моделирование переговоров с регулятором некорректно по существу. Ядерный регулятор — не «игрок» с рациональной стратегией, а государственный орган, действующий на основе фиксированных нормативных требований безопасности. Формализация регулятора как стратегического противника может привести к контрпродуктивным решениям и подрыву доверия.

**C-1-03** — ID идеи: R2-3-07  
ESG-критерии и углеродный след — механическое заимствование из корпоративной повестки. АЭС сама по себе низкоуглеродный источник; задача сооружения — ввести её в срок. Выбор схемы охлаждения определяется проектными условиями, а не оптимизацией углеродного следа.

**C-1-04** — ID идеи: R2-7-05  
Автоматизированная генерация компенсирующих мероприятий через TGN переоценивает возможности нейросетей. Изменение технологии работ на АЭС требует учёта физических ограничений, пересогласования с регулятором, изменения ПОС. Риск некритичного принятия рекомендаций нейросети.

**C-1-05** — ID идеи: R2-5-16  
Энергоэффективность здания АЭС не является определяющим фактором для проектирования и графика сооружения. Проектные решения определяются требованиями ядерной безопасности, радиационной защиты и сейсмостойкости.

**C-1-06** — ID идей: R2-1-01, R2-3-09, R1-7-07  
Claim-менеджмент на основе модели: арбитражные суды принимают фактические данные, а не вероятностные распределения. Оппонент может оспорить входные параметры. Без устоявшейся практики признания таких моделей их доказательная сила невысока.

**C-1-07** — ID идей: R2-6-09, R2-6-10  
Кибербезопасность и управление ИС выходят за пределы применимости модели Дмитришина. Для этих областей существуют специализированные методологии (ISO 27001, NIST CSF). PERT/GERT/Monte Carlo не предназначены для анализа киберугроз.

**C-1-08** — ID идеи: R2-3-06  
Agent-based моделирование подрядчиков основано на допущении о рациональном агенте. Калибровка требует данных, которые подрядчики не раскрывают. Без калибровки модель отражает допущения создателей, а не реальное поведение.

**C-1-09** — ID идеи: R2-5-11  
РАО на этапе строительства АЭС (до загрузки топлива) не образуются. Строительные отходы не являются радиоактивными. Включение в модель неоправданно.

**C-1-10** — ID идей: R2-3-02, R2-1-04  
Автоматическая синхронизация BIM с вероятностной моделью и «мгновенный» пересчёт Monte Carlo при каждом изменении: вычислительная перегрузка или потеря точности. Создание онтологии связей BIM↔график — самостоятельная масштабная задача.

**C-1-11** — ID: Кластер K-18 в целом  
Бесконтрольное расширение пространства атрибутов «на всё» (ESG, культура безопасности, метрология, кибербезопасность, ИС) ведёт к проклятию размерности, размыванию фокуса модели и утрате практической применимости.

**C-1-12** — ID идей: R1-1-09, R1-4-13, R1-1-15  
Шаг ввода энергоблоков определяется контрактными обязательствами, структурой финансирования и политическими решениями, а не результатами математической оптимизации. Практическая применимость ограничена.

**C-1-13** — ID идеи: R2-2-03  
Контур «автоснятия факта» из цифровых следов предполагает уровень цифровизации, отсутствующий на большинстве текущих проектов. Внедрение — отдельный масштабный проект цифровой трансформации.

**C-1-14** — ID идеи: R1-2-15  
Размещение персонала определяется градостроительными ограничениями, позицией местных властей, земельным законодательством — факторами, не поддающимися математической оптимизации.

**C-1-15** — ID идеи: R2-5-20  
Требования по защите от внешних воздействий — бинарные нормативные ограничения, а не точка в пространстве оптимизации. Включение в многокритериальную оптимизацию методологически некорректно.

**C-1-16** — ID идеи: R2-8-03  
Прогнозирование ликвидности подрядчиков: подрядчики не предоставляют детальную финансовую информацию. Без данных ML-модель будет работать на домыслах.

### Участник 2 (15 замечаний)

**C-2-01** — ID идеи: R1-2-01  
Сквозная PERT→GERT сеть рискует стать чрезмерно крупной и дорогой в сопровождении; без жёсткой дисциплины WBS и обновления факта модель быстро деградирует.

**C-2-02** — ID идеи: R1-2-06  
«Вероятностный барьер готовности» может превратиться в формальный чек-лист без прозрачного определения входных данных и ответственности за качество статусов.

**C-2-03** — ID идеи: R1-2-03  
GERT-модель лицензирования требует статистики по конкретному регулятору; при её отсутствии вероятности будут экспертными и оспоримыми.

**C-2-04** — ID идеи: R2-1-05  
«Оптимизация взаимодействия с регулятором» может переоценить управляемость процесса и предложить стратегии, неприемлемые с точки зрения комплаенса.

**C-2-05** — ID идеи: R1-2-04  
ML-прогноз циклов согласования РД уязвим к «смене процесса» (реорганизация, новый EDMS) — модель теряет переносимость между проектами.

**C-2-06** — ID идеи: R2-5-07  
Интеграция с EDMS даёт ценность только при высокой полноте метаданных; в реальности данные «грязные», прогноз нестабилен.

**C-2-07** — ID идеи: R2-2-07  
Учёт мощностей поставщиков упирается в конфиденциальность данных (портфель заказов завода, реальные слоты).

**C-2-08** — ID идеи: R2-5-06  
Моделирование устойчивости цепочки поставок может «взорваться» по числу сценариев; без строгого отбора top risk drivers система непрактична.

**C-2-09** — ID идеи: R2-1-01  
Для claim-менеджмента модель должна быть юридически защищаемой: воспроизводимость расчётов, согласованные допущения, audit trail. Иначе контрагент атакует методологию.

**C-2-10** — ID идеи: R2-3-02  
«Автоматическая синхронизация с цифровым двойником»: разная гранулярность BIM/4D vs ИКГ, несовместимые справочники, отсутствие единого master-data.

**C-2-11** — ID идеи: R2-2-05  
System completion / turnover — не только сеть работ, но и организационно-документный контур; без стандартизации границ систем подсеть станет источником споров.

**C-2-12** — ID идеи: R2-1-08  
Оптимизация бетонных работ через вероятностные длительности может не поймать физику процесса (окна бетонирования, уход за бетоном, ограничения по опалубке/кранам).

**C-2-13** — ID идеи: R2-3-10  
Hold-points/инспекции зависят от доступности инспектора (внешняя очередь); оптимизация даст «идеальный график», который рушится из-за внешних календарей.

**C-2-14** — ID идеи: R2-2-03  
Автоснятие факта из цифровых следов может давать систематические ошибки (заполнение задним числом, разная интерпретация статусов).

**C-2-15** — ID идеи: R2-2-04  
Оптимизация буферов как переменных требует согласования «что такое буфер» в контуре управления (график, договоры, резервы ресурсов).

### Участник 3 (12 замечаний)

**C-3-01** — ID идеи: R2-3-01  
Иерархическая оптимизация портфеля требует централизованного контроля над ресурсами всех дирекций, что противоречит организационной структуре и культуре автономии проектов.

**C-3-02** — ID идеи: R2-3-02  
Интеграция с BIM/PIM предполагает зрелый уровень цифровизации, которого нет. Автоматическая синхронизация в реальном времени потребует лет инвестиций.

**C-3-03** — ID идеи: R1-3-06  
Сквозная модель двух периодов игнорирует принципиально разный характер неопределённости: внешняя регуляторная (подготовительный) vs. внутренняя производственная (основной).

**C-3-04** — ID идеи: R1-3-05  
Anomaly detection на реальных данных стройплощадок будет генерировать огромное количество ложных срабатываний из-за фрагментарности и ручного ввода данных.

**C-3-05** — ID идеи: R1-3-02  
GERT для выбора контрактной стратегии: стратегия часто диктуется заказчиком, банками и политическими соглашениями, делая математическую оптимизацию вторичной.

**C-3-06** — ID идеи: R1-1-19  
Модель выбора ценообразования EPC не учитывает асимметрию информации и политические факторы переговоров. Решение принимается на уровне руководства по иным соображениям.

**C-3-07** — ID идеи: R2-3-03  
Bayesian updating learning curve: на экспортных проектах эффект обучения обнуляется из-за смены подрядчиков, локализации и разных регуляторных требований.

**C-3-08** — ID идеи: R2-3-04  
Game-theoretic: решения регулятора определяются политикой, сменой руководства и международным давлением, а не ожидаемой полезностью.

**C-3-09** — ID идеи: R2-1-01  
Monte Carlo прогнозы и ML-модели практически никогда не принимаются судами как достаточное доказательство причинно-следственной связи и размера убытков.

**C-3-10** — ID идеи: R1-3-10  
Интеграция модели в стандарт подготовительного периода требует перестройки процессов и регламентов Росатома; без корпоративной воли модель останется параллельным инструментом.

**C-3-11** — ID идеи: R1-1-03  
Оптимизация инвестиций недооценивает политический характер решения о старте. Объём вложений определяется необходимостью продемонстрировать прогресс.

**C-3-12** — ID идеи: R2-2-01  
Учёт корреляций и common-cause требует данных, которых в Росатоме пока недостаточно. Без них корреляционная структура будет вымышленной.

### Участник 4 (20 замечаний)

**C-4-01** — ID идеи: R1-1-01  
Для полноценной реализации требуется высокая детализация и достоверные распределения. На начальном этапе данных нет — риск «garbage in, garbage out».

**C-4-02** — ID идеи: R1-1-02  
Принципиальная непредсказуемость внешних процессов (лицензирование, общественные слушания) делает результаты Monte Carlo малоинформативными.

**C-4-03** — ID идеи: R1-1-03  
Оценки вероятностей состояний природы (отмена проекта, успех лицензирования) субъективны. Результат чувствителен к допущениям. Необходим анализ чувствительности.

**C-4-04** — ID идеи: R1-1-04  
Для уникального оборудования история поставок может отсутствовать, а длительности зависят от загрузки производителя, которая динамически меняется.

**C-4-05** — ID идеи: R1-1-06  
Адаптивное управление с ежемесячным пересчётом требует оперативных и достоверных данных. Данные с площадки поступают с задержкой, могут быть искажены. Необходим контур валидации.

**C-4-06** — ID идеи: R1-1-07  
ПНР зависят от результатов испытаний с неожиданными доработками. Модель должна включать GERT-циклы, что не конкретизировано.

**C-4-07** — ID идеи: R1-2-01  
Разная природа работ подготовительного и основного периодов. Объединение может привести к доминированию подготовительной неопределённости. Требуется иерархический подход.

**C-4-08** — ID идеи: R1-2-03  
Для новой юрисдикции статистика отсутствует, поведение регулятора непредсказуемо. Экспертные вероятности могут быть ошибочными.

**C-4-09** — ID идеи: R1-2-05  
Многомерный event-time graph: огромный объём данных, вычислительные ресурсы, сложность интерпретации для принятия решений.

**C-4-10** — ID идеи: R1-2-08  
ML-прогнозирование lead time не учитывает структурные изменения (смена поставщика, санкции). Требуется механизм обнаружения concept drift.

**C-4-11** — ID идеи: R1-3-02  
Исход конкурсных процедур — не случайная величина с известным распределением. Модель может быть слишком абстрактной для реальных закупок.

**C-4-12** — ID идеи: R1-4-11  
Логистика тяжеловесных грузов: инфраструктура может быстро меняться (паводок), модель на исторических распределениях не предскажет форс-мажор.

**C-4-13** — ID идеи: R1-5-07  
Локальная оптимизация предполагает линейную целевую функцию. Ресурсы неделимы, ограничения нелинейны. Требуется дискретная оптимизация.

**C-4-14** — ID идеи: R1-6-04  
Связь «объём вложений → сокращение сроков» нелинейна и слабо предсказуема. Модель может дать ложную уверенность в существовании оптимума.

**C-4-15** — ID идеи: R1-7-01  
Процент РД как интегральный показатель слишком груб. Модель должна учитывать критичность конкретных отсутствующих комплектов.

**C-4-16** — ID идеи: R2-1-01  
Суды требуют документальных подтверждений, а не математических моделей. Вероятностные модели могут быть отвергнуты как не имеющие юридической силы.

**C-4-17** — ID идеи: R2-2-01  
Оценить корреляции между длительностями работ сложно. Модель может стать перепараметризованной. При корреляциях стандартный CPM теряет смысл.

**C-4-18** — ID идеи: R2-4-04  
Потребление зависит от темпов СМР с высокой неопределённостью. Порочный круг: для прогноза потребления нужен прогноз темпов, который зависит от наличия материалов.

**C-4-19** — ID идеи: R2-5-06  
Вероятности редких событий (банкротство, пожар, санкции) объективно оценить невозможно. Анализ наихудшего случая без вероятностей может быть надёжнее.

**C-4-20** — ID идеи: R2-8-02  
Портфельное управление через GNN: гигантская задача по унификации данных, сложность интерпретации, риск «чёрного ящика».

### Участник 5 (6 замечаний)

**C-5-01** — ID идеи: R2-1-04  
Масштаб проблемы: десятки тысяч коллизий в 3D-модели. Без фильтрации по критичности модель породит «шум». Требуется механизм приоритизации коллизий.

**C-5-02** — ID идеи: R2-2-06  
Асимметрия информации: подрядчик обладает скрытой информацией о производительности. Инсентивы могут привести к «гонке за бонусами» в ущерб качеству.

**C-5-03** — ID идеи: R2-3-02  
Заявление об «абсолютно точном прогнозе без участия человека» — опасное упрощение. BIM-модель отстаёт от факта на 2–4 недели. Требуется валидация актуальности.

**C-5-04** — ID идеи: R2-5-08  
Культура безопасности — качественный, субъективный показатель. Подрядчики могут «накручивать» метрики. Эмпирические данные по корреляции с инцидентами ограничены.

**C-5-05** — ID идеи: R2-7-05  
TGN может предложить математически оптимальное, но технологически невозможное решение. Жёсткие технологические последовательности, согласование с регулятором. Требуется фильтр допустимости.

**C-5-06** — ID идеи: R2-8-03  
Финансовое положение подрядчика может ухудшиться из-за внешних факторов. Подрядчики не желают раскрывать финансовую информацию. Без актуальных данных прогноз неточен.

### Участник 6 (6 замечаний)

**C-6-01** — ID идеи: R1-1-03  
Для принципиально новых проектов отсутствует релевантная статистика. Экспертные оценки субъективны. Критерии Вальда/Байеса превращают «количественное обоснование» в формализованную субъективность.

**C-6-02** — ID идеи: R1-3-04  
Переход от Beta к нормальным распределениям при n≥30 нарушает условия ЦПТ: работы не iid. Применение нормального распределения недооценивает «толстые хвосты».

**C-6-03** — ID идеи: R2-3-02  
Иерархическая байесовская калибровка: данные фрагментированы, несопоставимы между проектами. Агрегация приведёт к экологической ошибке. Байесовское обновление на некорректных априорах усилит систематическую ошибку.

**C-6-04** — ID идеи: R1-7-03  
Динамическая синхронизация СМБ: производственные циклы имеют инерцию (2–3 месяца на заготовки). GNN оптимизирует абстрактный граф, игнорируя технологические циклы производства.

**C-6-05** — ID идеи: R2-5-06  
Для геополитических рисков объективные вероятности не существуют (кейнсианская неопределённость). Альтернативные поставщики могут одновременно стать недоступными при системном кризисе. Ложное чувство защищённости.

**C-6-06** — ID идеи: R2-7-04  
GNN оценивает технологичность только по геометрии, игнорируя квалификацию бригады, инструмент, погоду на высоте, культуру подрядчика. Систематические ошибки.

### Участник 7 (11 замечаний)

**C-7-01** — ID идеи: R2-3-04  
Game-theoretic моделирование регулятора противоречит базовым принципам культуры ядерной безопасности. Регулятор не торгуется — он следит за выполнением требований. Риск потери доверия и отзыва лицензии.

**C-7-02** — ID идеи: R1-1-17  
Выборка проектов Росатома мала (несколько десятков блоков) и гетерогенна. ML-алгоритмы на таком наборе неизбежно переобучатся.

**C-7-03** — ID идеи: R2-4-04  
Бетонная смесь — скоропортящийся продукт (часы). Невозможно произвести впрок и хранить как «запас». Для бетона задача — логистика автобетоносмесителей и пропускная способность, а не складские запасы.

**C-7-04** — ID идеи: R2-5-06  
Рынок ключевого оборудования ядерного острова строго монополизирован. При сбое у ключевого поставщика переключиться на «альтернативного» невозможно — цикл переразмещения займёт годы.

**C-7-05** — ID идеи: R2-1-04  
Интеграция коллизий BIM как «узлов переделок» легализует выпуск бракованной РД. Коллизия детерминирована: она либо есть, либо нет. Цель BIM — 100% устранение коллизий до выдачи документации.

**C-7-06** — ID идеи: R1-4-11  
Маршрут транспортировки КТО (корпус реактора, 300+ тонн) проектируется годами. На этапе доставки альтернативных маршрутов не существует — маршрут строго один.

**C-7-07** — ID идеи: R2-3-06  
ABM подрядчиков: Росатом не имеет доступа к внутренней коммерческой информации иностранных субподрядчиков. Алгоритмическое наложение штрафов на базе ABM может привести к банкротству субподрядчика.

**C-7-08** — ID идеи: R2-6-02  
Модель Дмитришина не предназначена для расчёта концентрации загрязняющих веществ. Это требует пространственных физико-химических моделей (уравнения дисперсии, акустическое моделирование).

**C-7-09** — ID идеи: R1-4-15  
Единый интегральный показатель готовности маскирует нулевую готовность по критическому ограничению. Готовность должна оцениваться по векторному совпадению hard constraints, а не скаляризироваться.

**C-7-10** — ID идеи: R2-8-02  
Портфельное перераспределение между РФ и экспортом: разные юрлица, обособленные источники финансирования, таможенные барьеры. Невозможно перебросить ресурсы между проектами по решению алгоритма.

**C-7-11** — ID идеи: R1-6-09  
Перевод бригады на новый объект: переоформление нарядов-допусков, инструктажи, изучение новых ППР. Модель без учёта «бюрократического трения» сгенерирует нереализуемо частые переброски.

### Участник 8 (10 замечаний)

**C-8-01** — ID идеи: R1-1-01  
Rework-циклы делают граф циклическим. Стандартные PERT/CPM не работают с циклическими графами; GERT существенно усложняет модель и сбор данных.

**C-8-02** — ID идеи: R1-1-03  
Количественная оценка полезности u(Γ_i, s_j) для сложных исходов крайне субъективна. Результаты зависят от экспертных оценок, что делает модель уязвимой для манипуляций.

**C-8-03** — ID идеи: R1-7-03  
Затраты на «динамическую синхронизацию» (постоянное перепланирование) могут превысить экономию. Без автоматизированной диспетчеризации — хаос.

**C-8-04** — ID идеи: R1-8-04  
ML для обнаружения аномалий: мала и неоднородна обучающая выборка «нормального» хода. Модель одного проекта даст ложные срабатывания на другом.

**C-8-05** — ID идеи: R2-1-01  
Вероятностная модель Monte Carlo — не факт, а распределение. Противная сторона оспорит допущения и методологию. Для внутреннего анализа — да, как аргумент в суде — маловероятно.

**C-8-06** — ID идеи: R2-8-02  
Асимметрия информации и локальные стимулы: руководители проектов будут завышать потребности, чтобы «выиграть» ресурс. GNN будет оптимизировать на искажённых данных.

**C-8-07** — ID идеи: R1-2-15  
Модель не учитывает привлекательность региона и рынка труда. В конкурентном регионе даже лучшая СБИ не удержит персонал.

**C-8-08** — ID идеи: R2-3-06  
Реальное поведение подрядчика определяется неформальными факторами (отношения, ожидания будущих контрактов, корпоративная культура), не сводимыми к функции полезности.

**C-8-09** — ID идеи: R1-1-09  
Шаг ввода блоков часто определяется немоделируемыми внешними факторами (требования энергосистемы, госфинансирование, политические обязательства).

**C-8-10** — ID идеи: R2-2-08  
Ценность информации от изысканий трудно оценить до их проведения. Модель работает с ожидаемым снижением риска — экспертной оценкой. Субъективность входных данных.
