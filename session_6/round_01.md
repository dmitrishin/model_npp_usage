## Раунд 1 — Мои идеи

**ID: R1-1-01**
**Идея:** Применение модели как комплексной системы оптимизации интегрированного комплексного графика (ИКГ) основного периода сооружения АЭС — от первого бетона до физического пуска.
**Обоснование:** Основной период сооружения (54–96 месяцев) представляет собой сеть из тысяч взаимосвязанных работ (СМР, ТМР, ЭМР, монтаж, ПНР), где критический путь определяется тепломонтажными работами и сваркой трубопроводов. Модель Дмитришина позволяет: (1) формализовать ИКГ как DAG с вероятностными оценками длительностей (PERT → нормальное распределение по мере накопления данных); (2) выявить через Monte Carlo не только критический, но и субкритические пути, оценить вероятность завершения к директивному сроку; (3) применить GERT с деревьями решений в узлах ветвления (выбор технологии монтажа, очерёдность пусковых комплексов); (4) выполнить итеративную оптимизацию критического пути и локальную оптимизацию сегментов; (5) задействовать ML для прогнозирования длительностей на основании фактических данных с площадки. Это даёт руководству проекта инструмент для обоснованного принятия решений по управлению сроками с учётом стоимости, рисков и качества (многокритериальная оптимизация F(Γ)→min). Бизнес-ценность: сокращение сроков основного периода и снижение вероятности его превышения, что напрямую влияет на стоимость проекта (каждый месяц задержки — проценты по кредитам, простой ресурсов, штрафные санкции по контракту).

**ID: R1-1-02**
**Идея:** Применение модели для оптимизации графика и стратегии подготовительного периода сооружения АЭС — от принятия решения о начале работ до первого бетона.
**Обоснование:** Подготовительный период (2–8 лет) включает параллельные и взаимозависимые процессы: лицензирование, проектирование (ПД и задел РД), контрактацию ОДЦИ, инженерные изыскания, подготовку площадки, сооружение пионерной базы и СМБ, контрактацию подрядчиков, получение разрешений. Эти процессы формируют сложную сетевую модель с высокой неопределённостью (например, сроки лицензирования — от 0,5 до 6 лет по международному опыту, сроки согласования ИТТ с заказчиком, результаты изысканий). Модель позволяет: (1) через PERT и Monte Carlo оценить вероятностное распределение сроков до первого бетона; (2) через GERT моделировать альтернативные сценарии (разные последовательности получения лицензий, варианты контрактной стратегии); (3) через критерии принятия решений (Вальда, Байеса, Гурвица) обосновывать выбор стратегии подготовительного периода при разном уровне неопределённости; (4) через многокритериальную оптимизацию F(Γ)→min балансировать сроки, стоимость подготовки и риски начала основного периода с незрелым проектом. Бизнес-ценность: обоснованное определение оптимальной продолжительности подготовительного периода и объёма работ, выполняемых до первого бетона. По данным NEA, завершённость проекта к первому бетону — главный фактор итоговых сроков и стоимости сооружения (R²=0,815).

**ID: R1-1-03**
**Идея:** Применение модели для оптимизации стратегии инвестирования в подготовительный период (решение о распределении вложений между подготовительным и основным периодами).
**Обоснование:** Рекомендация С2 выделяет фундаментальное противоречие: увеличение вложений в подготовку снижает риски строительства, но увеличивает объём средств под риском потери при отмене проекта. Модель Дмитришина может быть применена к этой задаче как комплексная система: (1) сетевая модель всего проекта (подготовительный + основной период) позволяет количественно связать объём работ подготовительного периода с длительностью и стоимостью основного; (2) критерии принятия решений при неопределённости (Вальд — для максимальной устойчивости к худшему сценарию, Гурвиц — для баланса оптимизма/пессимизма, Байес — при наличии статистики по аналогам) позволяют формализовать выбор стратегии инвестирования; (3) Monte Carlo даёт распределение итоговой стоимости проекта при разных сценариях инвестирования в подготовку; (4) многокритериальная оптимизация с построением фронта Парето визуализирует компромиссы «объём вложений в подготовку — риск потери при отмене — итоговая стоимость проекта». Бизнес-ценность: обоснованное принятие стратегического решения, определяющего всю экономику проекта, вместо интуитивного или политически мотивированного выбора. Актуально и для российских проектов (государственное финансирование), и для экспортных (внешнее финансирование).

**ID: R1-1-04**
**Идея:** Применение модели для оптимизации графика и стратегии контрактации и поставок оборудования длительного цикла изготовления (ОДЦИ) и прочего оборудования.
**Обоснование:** Контрактация и поставка оборудования — один из критических факторов сроков сооружения. По данным из стандарта и рекомендаций, задержки поставок (из-за длительного согласования ИТТ, конкурсных процедур, согласования НМЦ, сроков изготовления) на всех проектах приводят к значительному отставанию ТМР и ЭМР. Модель позволяет: (1) формализовать полный цикл «разработка ИТТ → согласование → закупка → изготовление → доставка → входной контроль → монтаж» как сетевую модель с вероятностными длительностями каждого этапа; (2) через Monte Carlo оценить вероятность поставки в срок каждой критической позиции; (3) через GERT моделировать альтернативные варианты поставки (разные поставщики, локализация vs. импорт); (4) через итеративную оптимизацию критических путей определить оптимальную последовательность закупок с учётом привязки к графику СМР и этапам ПНР; (5) применить ML для прогнозирования сроков изготовления и поставки на основе данных о фактическом исполнении по предыдущим проектам и заказам. Бизнес-ценность: минимизация простоев монтажных бригад из-за отсутствия оборудования на площадке, обеспечение принципа just-in-time.

**ID: R1-1-05**
**Идея:** Применение модели для количественного анализа рисков (QRA) сроков и стоимости проекта сооружения АЭС с поддержкой принятия решений.
**Обоснование:** Стандарт TCM NC и рекомендация H2 требуют проведения количественного анализа рисков с применением вероятностных методов. Модель Дмитришина представляет собой готовую математическую основу для QRA: (1) PERT с вероятностными распределениями длительностей формирует основу сетевой модели; (2) Monte Carlo даёт распределение итоговых сроков и стоимости, оценку вероятности завершения к целевой дате P(D≤T*); (3) многомерное атрибутное пространство (время, стоимость, риск, качество) позволяет вести QRA одновременно по нескольким измерениям; (4) критерии принятия решений формализуют использование результатов QRA для выбора стратегии реагирования на риски (Вальд — для ядернобезопасных решений, Байес — при наличии надёжной статистики, Гурвиц — для адаптивного планирования); (5) ML позволяет калибровать модель по фактическим данным, повышая точность QRA по мере реализации проекта. Бизнес-ценность: обоснование резервов сроков и стоимости (contingency и management reserve), переход от детерминированных графиков и смет к вероятностным, что критически важно для утверждения Целевой и Предельной стоимости проекта.

**ID: R1-1-06**
**Идея:** Применение модели для адаптивного управления проектом сооружения АЭС в ходе основного периода (система реагирования в реальном времени).
**Обоснование:** По мере выполнения работ основного периода фактические данные накапливаются: реальные длительности, фактическая производительность, задержки поставок, результаты контроля качества. Модель предусматривает механизм адаптации: переход от Beta-распределений к эмпирическим/нормальным при n≥30, обучение ML-моделей на фактических данных. Это позволяет создать систему адаптивного управления: (1) ежемесячно обновлять вероятностную сетевую модель фактическими данными; (2) пересчитывать критический и субкритические пути, вероятности завершения к директивным срокам; (3) выявлять через ML аномалии (отклонения факта от прогноза) — раннее предупреждение о проблемах; (4) через кластеризацию (K-means, DBSCAN) группировать типовые ситуации задержек для выработки стандартных мер реагирования; (5) через итеративную оптимизацию предлагать конкретные управленческие решения по компенсации отставания (перераспределение ресурсов, изменение очерёдности работ). Бизнес-ценность: переход от реактивного управления («тушение пожаров») к проактивному прогнозному управлению, раннее обнаружение проблемных зон.

**ID: R1-1-07**
**Идея:** Применение модели для оптимизации последовательности и сроков этапов пусконаладочных работ (ПНР) и их интеграции с графиком СМР.
**Обоснование:** Этапы ПНР (А0–А4, Б, В, Г) представляют собой сложную сеть взаимозависимых работ, начинающихся ещё в ходе основного периода (с подэтапа А-0). Рекомендация G6 указывает на необходимость планирования «от обратного» — от даты ввода через этапы ПНР. Модель позволяет: (1) формализовать все этапы и подэтапы ПНР как часть общей сетевой модели с вероятностными длительностями; (2) через GERT моделировать ветвления (например, решение ГРП о сроках подэтапа А-2, необходимость дополнительных испытаний после ХГО); (3) через Monte Carlo оценить вероятностное распределение сроков до физического пуска; (4) через многокритериальную оптимизацию балансировать сроки ПНР с завершением СМР и поставками оборудования; (5) через рекурсивную оптимизацию «от обратного» определить необходимые сроки поставки оборудования и завершения СМР для обеспечения своевременного начала каждого этапа ПНР на постоянных (а не временных) схемах. Бизнес-ценность: минимизация задержек при вводе энергоблока в эксплуатацию, исключение дорогостоящего использования временных схем вместо постоянных.

**ID: R1-1-08**
**Идея:** Применение модели для оптимизации выпуска рабочей документации (РД) и синхронизации с графиками СМР и поставок.
**Обоснование:** По данным стандарта и рекомендаций (G4), задел РД к первому бетону должен составлять 40–50%, а график выпуска РД — один из самых ранних планировочных документов проекта, определяющий темп всех последующих процессов. Темп проектирования зависит от циклов согласования ИДП между смежными дисциплинами и от получения данных от поставщиков оборудования. Модель позволяет: (1) формализовать процесс выпуска РД как сетевую модель, включая взаимозависимости между дисциплинами (технологическая, строительная, электротехническая, вентиляционная) и циклы обмена данными с поставщиками; (2) через Monte Carlo оценить вероятность формирования необходимого задела РД к первому бетону; (3) через итеративную оптимизацию определить оптимальную последовательность выпуска комплектов РД с привязкой к пакетам работ СМР и графику поставок; (4) через ML прогнозировать фактические сроки выпуска комплектов на основе данных о производительности проектного института. Бизнес-ценность: исключение ситуаций, когда строительство готово к СМР, но нет РД, или когда РД выпущена, но закупка оборудования не начата.

**ID: R1-1-09**
**Идея:** Применение модели для обоснования и оптимизации продолжительности основного периода сооружения и шага ввода энергоблоков многоблочной АЭС.
**Обоснование:** Рекомендация G1 указывает, что ключевые параметры проекта — продолжительность основного периода (48, 54, 65, 72, 84, 96 месяцев) и шаг ввода блока (12, 18, 24 месяца) — должны иметь технико-экономическое обоснование. Модель позволяет: (1) для каждого варианта продолжительности основного периода построить вероятностную сетевую модель и оценить через Monte Carlo реалистичность заявленного срока; (2) через многокритериальную оптимизацию F(Γ)=(T,C,R,Q)→min определить Парето-оптимальные сочетания «длительность — стоимость — риск» для разных вариантов; (3) для многоблочной АЭС через расширенную сетевую модель определить оптимальный шаг ввода с учётом совместного использования ресурсов (краны, бригады, СМБ) между блоками; (4) через критерии принятия решений обосновать выбор конкретного варианта с учётом отношения к риску (Вальд для консервативного, Гурвиц для сбалансированного). Бизнес-ценность: предотвращение утверждения заведомо невыполнимых графиков, которые, по данным рекомендаций, «достаточно быстро теряют актуальность» и вызывают каскадные негативные эффекты вплоть до банкротства подрядчиков и судебных разбирательств.

**ID: R1-1-10**
**Идея:** Применение модели для оптимизации работы строительно-монтажной базы (СМБ) и синхронизации заготовительных и укрупнительных работ с графиком монтажа на площадке.
**Обоснование:** СМБ обеспечивает площадку укрупнёнными конструкциями, армоблоками, блоками трубопроводов, подготовленным электрооборудованием и другой продукцией. Темп работы СМБ должен быть синхронизирован с темпом монтажа на площадке. Модель позволяет: (1) формализовать производственные потоки СМБ (арматурное хозяйство, тепломеханический комплекс, электромонтажный комплекс и др.) как сеть взаимосвязанных операций с вероятностными длительностями; (2) через Monte Carlo оценить пропускную способность СМБ и вероятность обеспечения площадки укрупнёнными элементами в срок; (3) через локальную оптимизацию сегментов определить «узкие места» в производственных линиях СМБ; (4) через многокритериальную оптимизацию балансировать загрузку СМБ с потребностями нескольких объектов пускового комплекса одновременно. Бизнес-ценность: обеспечение поточного метода строительства, исключение простоев монтажных бригад из-за неготовности укрупнённых элементов, рациональная загрузка производственных мощностей базы.

**ID: R1-1-11**
**Идея:** Применение модели для оценки и оптимизации проектной зрелости (design maturity) к моменту первого бетона и прогнозирования её влияния на сроки и стоимость основного периода.
**Обоснование:** По данным NEA 7530, существует выраженная корреляция (R²=0,815) между процентом завершённости проекта на момент первого бетона и удельной стоимостью строительства. На Flamanville-3 лишь ~40% детального проекта было готово к первому бетону, что привело к 4500+ изменениям. Модель позволяет: (1) ввести «процент проектной зрелости» как атрибут в многомерном пространстве состояний проекта xi=(ti, ai); (2) на основе данных по реализованным проектам (включая корейскую серийную программу, Flamanville-3, Olkiluoto-3, проекты Росатома) обучить ML-модели для прогнозирования влияния уровня зрелости на итоговые сроки и стоимость; (3) через многокритериальную оптимизацию определить целевой уровень зрелости к первому бетону для конкретного проекта; (4) через критерии принятия решений (Вальд для безопасности, Гурвиц для баланса) формализовать момент перехода «готов / не готов к первому бетону». Бизнес-ценность: количественное обоснование решения о начале основного периода (Construction Readiness Review), предотвращение преждевременного старта строительства с недоработанным проектом — главной причины срывов сроков и бюджета по мировому опыту.

**ID: R1-1-12**
**Идея:** Применение модели для управления рисками экспортных проектов сооружения АЭС с учётом страновой специфики.
**Обоснование:** Экспортные проекты Росатома сопряжены с дополнительными рисками: регуляторные требования страны-заказчика, лицензирование в чужой юрисдикции, локализация, валютные риски, политические риски, квалификация местных подрядчиков, культурные особенности, импортные ограничения. Модель позволяет: (1) через GERT с вероятностями переходов моделировать альтернативные сценарии развития проекта, обусловленные страновыми факторами (задержка лицензирования, изменение требований регулятора, политическая нестабильность); (2) через критерии принятия решений формализовать выбор стратегии для разных уровней странового риска: Вальд для стран с высокой неопределённостью (максимальная устойчивость к худшему сценарию), Байес при наличии статистики по аналогичным проектам в регионе; (3) через Monte Carlo количественно оценить вероятность соблюдения контрактных сроков с учётом совокупности страновых рисков; (4) через ML на данных по реализованным экспортным проектам выявить закономерности и предикторы задержек. Бизнес-ценность: обоснованная оценка рисков для контрактных переговоров, обоснование рисковых надбавок и резервов, формирование адекватных ожиданий по срокам и стоимости экспортного проекта.
